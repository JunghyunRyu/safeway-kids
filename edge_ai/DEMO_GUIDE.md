# SafeWay Kids Edge AI PoC 데모 사용 가이드

## 시작하기

### 1단계: 서버 실행
```bash
cd edge_ai
python main_v2.py
```
"Running on http://127.0.0.1:7860" 메시지가 나올 때까지 약 5~8초 대기.

### 2단계: 브라우저 열기
`http://localhost:7860` 접속

### 3단계: 시작 화면
- 4개 도메인 카드 (엣지 컴퓨팅 / CCTV / 안면인식 / 사각지대)가 보임
- 하단의 상태 표시등 확인:
  - 🟢 카메라 연결 — 웹캠이 잡혀야 함
  - 🟢 모델 로드 — AI 모델이 준비됨
  - 🟡 백엔드 — standalone 모드 (정상)
- **[▶ 데모 시작]** 버튼 클릭

## 시나리오별 시연 방법

### ① 승하차 안면인식 (첫 화면)

**화면 구성:** 웹캠 영상 + 우측에 프레젠터 가이드 + 성능 대시보드

**시연 순서:**

1. **얼굴 등록** (먼저 해야 인식이 됩니다)
   - 우측 사이드바에서 "보호자 동의" 또는 "얼굴 등록" 버튼 클릭
   - 동의서 모달이 열림 → 체크박스 체크 → 보호자/원생 이름 입력
   - 웹캠 앞에 정면으로 서서 **[등록]** 클릭
   - "김민준 등록 완료" 메시지 확인
   - (좌15도, 우15도 추가 등록하면 정확도 향상)

2. **승차 인식 확인**
   - 등록 완료 후 웹캠 앞에 서면 자동으로 인식
   - **초록색 박스** + "김민준 탑승확인 (95%)" 표시
   - 미등록 얼굴은 **노란색 박스** + "미등록"
   - 하단 이벤트 로그에 기록 추가됨
   - 우측 성능 대시보드: CPU%, FPS, 추론시간(ms) 확인

3. **심사위원에게 설명할 포인트:**
   - "보호자 동의 절차를 거친 후에만 등록 가능합니다"
   - "원본 이미지는 저장하지 않고, 512차원 특징 벡터만 암호화 저장합니다"
   - "실시간 추론 시간이 100ms 이하입니다"

---

### ② 이상 행동 감지

**시나리오 전환:** 상단 스텝 위저드에서 **[② 이상 행동 감지]** 탭 클릭

**시연 순서:**

1. **정상 상태 확인**
   - 의자에 앉아 있으면 **초록색 박스** + "Normal" 표시
   - 프레젠터 가이드에 "정상 착석 상태" 안내

2. **이상 행동 유발**
   - 웹캠 앞에서 **천천히 일어서서 3초 이상 유지**
   - AI가 감지하면:
     - **빨간색 박스** + "STANDING" 표시
     - 화면 테두리가 빨갛게 변함
     - 음성: "경고: 이상 행동이 감지되었습니다"
     - 이벤트 로그에 기록

3. **다시 앉기**
   - 앉으면 다시 초록색 Normal로 돌아감

4. **심사위원에게 설명할 포인트:**
   - "2초 이상 지속될 때만 경고합니다 (거짓 경고 억제)"
   - "정차 중에는 서 있어도 경고하지 않습니다 (운행 모드 구분)"
   - "운전기사에게는 음성으로만 알려서 운전 방해를 최소화합니다"

---

### ③ 잔류 인원 감지

**시나리오 전환:** 상단에서 **[③ 잔류 인원 감지]** 탭 클릭

**시연 순서:**

1. **대기 상태**
   - 화면에 "대기 - 시동 OFF를 눌러주세요" 표시
   - 하단에 **[시동 OFF]** / **[시동 ON]** 버튼이 나타남

2. **잔류 인원 있는 상태에서 시동 OFF**
   - 웹캠 앞에 사람이 있는 상태에서 **[시동 OFF]** 클릭
   - AI가 사람을 스캔:
     - **빨간색 박스** + "잔류 인원 감지!" 경고
     - 음성: "잔류 인원이 감지되었습니다. 확인해 주세요"
     - 이벤트 로그에 기록

3. **잔류 인원 없는 상태에서 시동 OFF**
   - 웹캠에서 비켜서서 (사람 없는 상태) **[시동 OFF]** 클릭
   - **"ALL CLEAR"** 초록색 표시

4. **심사위원에게 설명할 포인트:**
   - "시동을 끄면 자동으로 5프레임을 연속 분석합니다"
   - "차량 구석에 숨어있는 아이도 감지합니다"
   - "감지 시 관리자와 학부모에게 동시 알림이 갑니다"

---

### ④ 사각지대 감지

**시나리오 전환:** 상단에서 **[④ 사각지대 감지]** 탭 클릭

**화면 전환:** 웹캠 영상 대신 **Bird's Eye View (조감도)** Canvas가 나타남

**시연 순서:**

1. **시나리오 선택**
   - 하단 드롭다운에서 시나리오 선택:
     - "후방 접근" — 어린이가 뒤에서 다가옴
     - "측면 사각지대" — 옆에서 갑자기 나타남
     - "복수 어린이" — 3명이 동시 접근

2. **[▶ 재생] 클릭**
   - 가상 어린이(원)가 차량(직사각형)에 접근
   - **LiDAR 360도 레이** (방사형 파란 선)가 어린이를 감지
   - **초음파 센서 콘** (반투명 부채꼴)이 표시
   - 거리에 따라 3단계 색상 변화:
     - 🟡 4m 이내: **주의** (노란색)
     - 🟠 2.5m 이내: **경고** (주황색)
     - 🔴 1.0m 이내: **위험** (빨간색 + 경고음)

3. **마우스 모드** (인터랙티브)
   - **[마우스 모드]** 버튼 클릭
   - 마우스로 어린이 점을 드래그하여 직접 이동
   - 차량에 가까이 가져가면 경고 발생

4. **심사위원에게 설명할 포인트:**
   - "VLP-16 LiDAR 360도 + 초음파 12개 센서를 시뮬레이션합니다"
   - "센서 노이즈(±3cm)와 드롭아웃(2%)까지 반영한 물리 기반 시뮬레이션입니다"
   - "두 센서를 7:3 비율로 융합하여 오탐을 줄입니다"
   - "실제 하드웨어 장착 시 동일한 알고리즘이 작동합니다"

## 추가 기능

### 현장 UI 프리뷰
- 우측 상단 **[현장 UI]** 버튼 클릭
- **인솔교사 뷰**: 큰 버튼 3개 (승차/운행/하차) + 승하차 명단
- **운전기사 뷰**: 신호등(🟢🟡🔴) + 음성 경고만
- "실제 차량에서는 이 화면이 태블릿에 표시됩니다"

### 성능 대시보드
- 우측 사이드바에 항상 표시:
  - CPU 사용률 (%)
  - 메모리 (MB)
  - FPS (프레임/초)
  - 추론 시간 (ms)
- "i5 CPU에서 이 정도이고, Jetson에서는 8~10배 빨라집니다"

### 데모 리셋
- 우측 상단 **[데모 리셋]** 버튼 → 모든 상태 초기화

## 문제 해결

| 증상 | 해결 |
|------|------|
| 웹캠 화면 안 나옴 | 다른 프로그램이 웹캠 사용 중인지 확인. Zoom/Teams 종료 |
| 얼굴 인식 안 됨 | 조명 확인. 얼굴이 정면이어야 함. 등록을 먼저 해야 함 |
| FPS가 너무 낮음 | 다른 프로그램 종료. 자동으로 해상도 축소됨 |
| 소리가 안 남 | 브라우저 음소거 확인. "데모 시작" 버튼을 클릭해야 음성 활성화 |
| 서버 접속 안 됨 | `python main_v2.py` 실행 확인. 포트 7860 확인 |
