# Phase 0 — Intake: 플랫폼 운영자 대시보드

**작성일:** 2026-03-20

---

## 1. Requirement Brief

플랫폼 운영자(platform_admin)가 전체 시스템을 관리할 수 있는 웹 대시보드가 필요하다. 현재 `web/` 대시보드는 학원 관리자(academy_admin) 전용으로, 플랫폼 운영자가 사용하면 `/academies/mine` 호출 시 데이터가 없어 빈 화면이 표시된다.

기존 `web/` 앱을 역할 기반으로 확장하여, platform_admin 로그인 시 전체 시스템 관리 기능을 제공한다.

---

## 2. Goals / Non-goals

### Goals
| ID | 목표 |
|----|------|
| G1 | platform_admin 역할 기반 대시보드 (전체 학원/사용자/차량 통계) |
| G2 | 사용자 관리 (전 역할 계정 생성/조회) |
| G3 | 학원 관리 (학원 생성/목록/수정) |
| G4 | 전체 차량 관리 (크로스 학원 차량 목록) |
| G5 | 전체 청구 관리 (크로스 학원 청구서 조회/정산) |
| G6 | 엑셀 업로드 UI (학원 선택 → 학생 데이터 업로드) |
| G7 | 컴플라이언스 문서 관리 UI (업로드/만료 조회) |
| G8 | 테스트 시드 데이터 생성 (백엔드 API + UI 버튼) |

### Non-goals
- 모바일 앱 변경
- 새 웹 앱 생성 (기존 web/ 확장)
- 실시간 차량 맵 (관제센터는 다음 단계)

---

## 3. Assumption Register

| ID | 가정 |
|----|------|
| A1 | 기존 web/ 앱에 역할 분기를 추가하여 확장한다 |
| A2 | 백엔드에 사용자 관리 API가 추가 필요하다 (현재 없음) |
| A3 | dev-login으로 platform_admin 역할 테스트가 가능하다 |
| A4 | 시드 데이터 생성은 백엔드 API 엔드포인트로 제공한다 |

- [ ] AC-3: 전 역할 사용자 목록 조회/생성 가능
- [ ] AC-4: 전체 차량 목록 조회/등록 가능
