# Phase 0 — Intake: 웹 대시보드 100% 완성

**작성일:** 2026-03-20
**현재 점수:** 41/100
**목표 점수:** 90+/100

---

## 1. Requirement Brief

웹 대시보드를 프로토타입 수준(41점)에서 실운영 가능한 프로덕션 수준(90+점)으로 끌어올린다. 감사 결과 도출된 6개 카테고리의 핵심 결함을 체계적으로 수정한다.

---

## 2. Goals (우선순위 순)

### P0 — 즉시 수정 (가장 큰 점수 향상)
| ID | 목표 | 현재 | 목표 점수 기여 |
|----|------|------|-------------|
| G1 | 전 엔티티 Edit/Update 기능 | 0% | +20점 |
| G2 | 전 엔티티 Delete(소프트) 기능 | 0% | +10점 |
| G3 | 확인 다이얼로그 (삭제/결제/파이프라인 등) | 없음 | +5점 |
| G4 | 폼 유효성 검증 + 구체적 에러 메시지 | alert()만 | +8점 |

| ID | 목표 | 현재 | 목표 점수 기여 |

## 3. Non-goals
- 실시간 차량 맵 (별도 마일스톤)
- 다크 모드
- 키보드 단축키
- 드래그앤드롭 스케줄 빌더

## 4. Acceptance Criteria
- [ ] AC-1: 모든 엔티티(학생, 차량, 학원, 사용자, 요금제)에 Edit 기능
- [ ] AC-2: 모든 엔티티에 Delete(비활성화) 기능 + 확인 다이얼로그
- [ ] AC-3: 모든 폼에 클라이언트 유효성 검증 + 필드별 에러 메시지
- [ ] AC-4: 테이블 페이지네이션 (20행/페이지)
- [ ] AC-5: CSV 내보내기 (학생, 청구서, 사용자)
- [ ] AC-6: 학생/청구서 상세 보기 모달
- [ ] AC-7: 대시보드 KPI 시각화 (막대/원형 차트)
- [ ] AC-8: 공통 DataTable + FormModal 컴포넌트
- [ ] AC-9: 확인 다이얼로그 (삭제, 결제, 파이프라인)
- [ ] AC-10: 기존 테스트 유지 + 신규 테스트 추가
- [ ] AC-11: TypeScript 0 오류
