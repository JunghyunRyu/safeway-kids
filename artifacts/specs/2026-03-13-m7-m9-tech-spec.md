# M7-M9 Final Tech Spec

## M7: Billing System

### Problem
학원-학부모 간 셔틀 이용료 정산 시스템이 없어 수익 모델 부재.

### Goals
- 학생별 월간 이용 횟수 자동 집계
- 월별 청구서(Invoice) 자동 생성
- 학원별 요금제(Plan) 관리
- 학부모 결제 내역 조회 (모바일)

### Non-goals
- 실제 PG 결제 연동 (MVP에서는 청구서 생성까지만)
- 거리 가중 요금 (추후)

### Models
- **BillingPlan**: academy_id, name, price_per_ride, monthly_cap, is_active
- **Invoice**: parent_id, academy_id, student_id, billing_month, total_rides, amount, status(pending/paid/overdue), due_date
- **Payment**: invoice_id, amount, method, paid_at, transaction_ref

### APIs
- POST /billing/plans (admin)
- GET /billing/plans?academy_id= (admin)
- POST /billing/generate-invoices?month=2026-03 (admin, batch)
- GET /billing/invoices/my (parent)
- GET /billing/invoices?academy_id= (admin)
- POST /billing/invoices/{id}/mark-paid (admin)

### Mobile
- 학부모 앱: "결제" 탭 추가 → 월별 청구서 목록, 상세

## M8: Academy Admin Web Dashboard

### Problem
학원 관리자가 학생/스케줄/차량/정산을 관리할 UI가 없음.

### Goals
- React SPA (Vite + TypeScript)
- 기존 백엔드 API 활용
- 학생 목록/등록, 스케줄 관리, 차량 현황, 청구서 확인

### Pages
1. Login (dev-login 활용)
2. Dashboard (오늘 현황 요약)
3. Students (목록, 검색, 등록)
4. Schedules (템플릿 관리, 일별 현황)
5. Vehicles (차량 목록, 배정 현황)
6. Billing (청구서 목록, 생성, 정산)

### New Backend APIs
- GET /academies/mine (현재 관리자의 학원 조회)
- GET /students?academy_id= (학원별 학생 목록)

## M9: Safety Escort Matching

### Problem
세림이법에 따른 안전도우미 매칭 마켓플레이스 부재.

### Goals
- 안전도우미 가용시간 등록
- 자동/수동 매칭
- 출근 확인

### Models
- **EscortAvailability**: escort_id, date, start_time, end_time, status(available/matched/completed)
- **EscortShift**: escort_id, vehicle_assignment_id, date, check_in_at, check_out_at, compensation_amount

### APIs
- POST /escorts/availability (escort)
- GET /escorts/availability/my (escort)
- POST /escorts/match (admin, auto-match)
- POST /escorts/shifts/{id}/check-in (escort)
- POST /escorts/shifts/{id}/check-out (escort)
- GET /escorts/shifts/my (escort)
