# Requirement Brief: Design System Redesign + Admin/Student Screens

**Date:** 2026-03-17
**Milestone:** Design System v2 + Role Expansion
**Status:** Phase 0 — Intake

---

## Goals

1. **색깔 톤 수립** — safeway-kids 전용 독창적 색상 팔레트 설계 (흔하지 않은 톤)
2. **디자인 시스템 구축** — 중앙화된 테마/색상 상수 파일 생성 및 전체 앱 적용
3. **신규 역할 메뉴** — 관리자(admin), 학생(student) 화면 및 탭 네비게이션 추가
4. **UI/UX 개선** — Schedule, 청구서(Billing), 내 정보(Profile) 화면의 프로덕션 품질 향상
5. **모바일 앱 전체 색상 적용** — 하드코딩된 색상을 모두 테마 상수로 교체
6. **랜딩 사이트 색상 동기화** — site/의 Tailwind 토큰을 새 팔레트로 업데이트

## Non-Goals

- 백엔드 API 신규 엔드포인트 추가 (기존 API 재활용)
- Kakao Map 연동 방식 변경
- WebSocket 트래킹 로직 변경
- 새로운 인증/인가 방식 도입

## Context

### 대상 사용자
| 역할 | 설명 |
|------|------|
| 학부모 (parent) | 자녀를 등하원시키는 학부모 |
| 기사 (driver) | 차량 운전 기사 |
| 안전도우미 (safety_escort) | 동승 안전 요원 |
| **관리자 (admin)** | 학원/플랫폼 운영 관리자 (신규) |
| **학생 (student)** | 탑승 학생 본인 (신규) |

### 현재 문제점
1. **색상**: 범용적인 `#2196F3` 파란색 → 독창성 없음
2. **색상 관리**: 모든 컴포넌트에 하드코딩 → 유지보수 불가
3. **Schedule**: 기본 목록 나열 → UX 정보 계층 없음
4. **Billing**: 청구서 카드 정보 불충분 → 실제 사용 불가 수준
5. **Profile**: 텍스트 나열 + 단순 로그아웃 버튼 → 테스트용 수준
6. **관리자/학생 화면**: 완전 부재

## Assumption Register

| ID | 가정 내용 |
|----|----------|
| A1 | 관리자는 기존 academy_admin 또는 platform_admin 역할에 매핑 |
| A2 | 학생 화면은 자신의 오늘 일정, 기사 정보, 간단한 프로필로 구성 |
| A3 | 신규 화면에 필요한 API는 기존 엔드포인트 조합으로 구성 가능 |
| A4 | 색상 팔레트는 WCAG 2.1 AA 대비율(4.5:1 이상)을 준수해야 함 |
| A5 | 한국어 UI 유지 (i18n 구조 확장) |
| A6 | 랜딩 사이트 색상도 동일 팔레트 적용 |
