## 1. Requirement Brief

### 1.1 프로젝트 개요

SafeWay Kids는 어린이 통학차량 안전 플랫폼이다. 본 PoC는 **규제 샌드박스 심사위원**에게 AI 기반 안전 기술의 논리적 타당성을 입증하기 위한 **소프트웨어 시뮬레이션 데모**이다.

- **실행 환경:** Windows i5 PC (고가 하드웨어 없이 일반 사무용 PC에서 실행)
- **입력 장치:** 내장/USB 웹캠 1대
- **출력:** 웹 브라우저 기반 실시간 데모 UI
- **대상:** 비기술 심사위원이 직관적으로 이해할 수 있는 시각적 완성도 필수
- **기존 코드:** `edge_ai/` 디렉토리에 Flask+MJPEG 기반 3-시나리오 프로토타입 존재 (참고용, 완전 재작성 가능)

### 1.2 4개 도메인별 요구사항

#### 도메인 1: 엣지 컴퓨팅 디바이스 (Edge Computing Device) 시뮬레이션

**실제 목표 하드웨어:** NVIDIA Jetson Orin Nano / Jetson AGX Xavier
**PoC 시뮬레이션 방식:** i5 CPU + YOLOv8n (CPU 추론)으로 Jetson급 추론 파이프라인 재현

| 항목 | 요구사항 | 비고 |
|------|----------|------|
| AI 추론 엔진 | YOLOv8n (Ultralytics) + MediaPipe | CPU 모드, PyTorch CPU 빌드 |
| 추론 지연시간 | 프레임당 200ms 이하 (5 FPS 이상) | 신청서 목표치 반영 |
| 멀티 모델 동시 운용 | 객체 감지 + 포즈 추정 + 안면 감지 파이프라인 | 순차 실행 허용 (PoC) |
| 리소스 모니터링 | CPU/메모리 사용률 실시간 표시 | 데모 신뢰성 입증 |
| 추론 성능 대시보드 | 모델별 추론 시간, FPS, 정확도 시각화 | 심사위원용 |

**UI 표현:**
- 화면 상단에 "Edge Device Status" 패널: CPU 사용률, 메모리, 추론 지연시간(ms), FPS 실시간 표시
- Jetson 디바이스를 시뮬레이션하고 있음을 명시하는 인포그래픽

#### 도메인 2: 영상 입력 장치 — CCTV 시뮬레이션 (Vehicle Interior Camera)

**실제 목표 하드웨어:** 차량 내부 광각 카메라 (170도 FOV)
**PoC 시뮬레이션 방식:** USB 웹캠으로 차량 내부 카메라 역할 대행

| 항목 | 요구사항 | 비고 |
|------|----------|------|
| 영상 입력 | USB 웹캠 실시간 스트리밍 (640x480) | OpenCV VideoCapture |
| 이상 행동 감지 | YOLOv8 + MediaPipe Pose로 서기/넘어짐 감지 | 시나리오 ② |
| 잔류 인원 감지 | YOLOv8 person detection + 시동 OFF 트리거 | 시나리오 ③ |
| 영상 녹화 시뮬레이션 | 이벤트 발생 시 스냅샷 저장 (JPEG) | 증거 보존 시뮬레이션 |
| 야간 모드 시뮬레이션 | 적외선 카메라 효과 (그레이스케일 + 밝기 보정) | 24시간 운영 입증 |

**시나리오 ② — 운행 중 이상 행동 감지:**
1. 웹캠이 "차량 내부"를 촬영
2. YOLOv8n으로 사람(person) 감지
3. MediaPipe PoseLandmarker로 자세 분석
4. 어린이가 일어나거나 넘어지면 → 빨간 바운딩 박스 + "위험: 이동 금지" 경고
5. 이벤트 로그 기록 + 관리자 알림

**시나리오 ③ — 시동 OFF 잔류 인원 감지:**
1. 사용자가 "시동 OFF" 버튼 클릭
2. YOLOv8n으로 차량 내 잔류 인원 스캔 (multi-frame 분석)
3. 인원 감지 시 → "잔류 인원 감지" 음성 경고 + 관리자 경보 + 학부모 푸시
4. 미감지 시 → "ALL CLEAR" 표시

#### 도메인 3: 안면 인식 터미널 (Face Recognition Terminal) 시뮬레이션

**실제 목표 하드웨어:** 스마트 미러 / 태블릿 형태 안면 인식 단말기
**PoC 시뮬레이션 방식:** 웹캠 + MediaPipe FaceDetection + 히스토그램 기반 매칭 (또는 deep embedding)

| 항목 | 요구사항 | 비고 |
|------|----------|------|
| 안면 감지 | MediaPipe FaceDetector (TFLite) | 실시간 바운딩 박스 |
| 안면 인식 | 등록된 얼굴과 매칭 → 신원 확인 | 시나리오 ① |
| 인식 성공률 | 95% 이상 (정면, 정상 조명 기준) | 신청서 목표치 |
| 마스크 착용 대응 | 마스크 착용 시에도 70% 이상 인식 | 신청서 반영 |
| 얼굴 등록 | 실시간 웹캠 캡처로 새 얼굴 등록 | 데모 중 즉석 등록 |
| 비접촉 인증 | 카메라 앞에 서면 자동 인식 (터치 불필요) | 위생/편의성 |
| 조명 보정 | 히스토그램 평활화 등 전처리 | 다양한 조명 대응 |

**시나리오 ① — 안면 인식 승하차:**
1. 데모 시작 전 원생 얼굴 등록 (이름 입력 + 웹캠 캡처)
2. 원생이 카메라 앞에 서면 자동 감지
3. 등록된 얼굴과 매칭 → "OOO 원생 탑승 확인" 표시
4. 학부모 앱으로 푸시 알림 전송 (백엔드 연동)
5. 미등록 얼굴 → "미등록" 표시 (노란색)

**인식 정확도 향상 방안:**
- 현재 구현: 히스토그램 기반 특징 매칭 (단순하지만 조명에 민감)
- 개선 옵션 A: `face_recognition` 라이브러리 (dlib 기반 128-dim embedding) — dlib 빌드 이슈 가능
- 개선 옵션 B: InsightFace / ArcFace (ONNX Runtime) — 높은 정확도, CPU에서도 가능
- 개선 옵션 C: MediaPipe FaceMesh + custom embedding — 가벼운 대안

#### 도메인 4: 사각지대 감지 센서 (Blind Spot Detection Sensor) 시뮬레이션

**실제 목표 하드웨어:** LiDAR 센서 + 초음파 센서 (차량 전후방)
**PoC 시뮬레이션 방식:** 소프트웨어로 센서 데이터를 생성하고 시각화

| 항목 | 요구사항 | 비고 |
|------|----------|------|
| LiDAR 시뮬레이션 | 가상 포인트 클라우드 생성 + 2D/3D 시각화 | Canvas or Three.js |
| 초음파 센서 시뮬레이션 | 거리 기반 접근 감지 (근접 경고) | 시각적 게이지 표시 |
| 센서 융합 (Sensor Fusion) | LiDAR + 초음파 데이터 결합 의사결정 | 알고리즘 시뮬레이션 |
| 차량 조감도 (Bird's Eye View) | 차량 주변 360도 감지 범위 시각화 | 2D 탑뷰 UI |
| 어린이 접근 경고 | 감지 영역 내 객체 진입 시 단계별 경고 | 3단계: 주의/경고/위험 |
| 시나리오 재생 | 사전 정의된 시나리오 자동 재생 | 데모 안정성 |

**시나리오 ④ — 사각지대 어린이 감지:**
1. 차량 조감도(Bird's Eye View) UI에 차량 + 감지 범위 표시
2. 시뮬레이션된 "어린이 객체"가 차량 후방에 접근
3. 거리에 따라 단계별 경고:
   - 5m 이내: 주의 (노란색)
   - 3m 이내: 경고 (주황색)
   - 1.5m 이내: 위험 (빨간색 + 경고음)
4. LiDAR 포인트 클라우드에 어린이 형태 하이라이트
5. 초음파 거리 게이지 실시간 업데이트
6. "접근 어린이 감지 — 차량 이동 금지" 경고 팝업 + 관리자 알림

**시뮬레이션 구현 방안:**
- **옵션 A (권장):** Python 기반 시뮬레이션 엔진 + 웹 UI (Canvas 2D)
  - 가상 차량 모델 (직사각형) 중심, 전후좌우 감지 영역 표시
  - 마우스/자동으로 "어린이 객체" 이동
  - LiDAR 레이 캐스팅 시뮬레이션 (방사형 레이)
  - 초음파 콘 영역 시각화
- **옵션 B:** Three.js 3D 시각화 (더 인상적이나 복잡도 높음)
- **옵션 C:** 사전 녹화 영상 + 오버레이 (가장 안정적이나 인터랙티브하지 않음)

## 2. Goals / Non-goals

### Goals (이 PoC가 반드시 달성해야 할 것)

1. **G1 — 4개 시나리오 완전 시연:** 안면인식 승하차, 이상행동 감지, 잔류인원 감지, 사각지대 감지 4가지 모두 동작하는 데모
2. **G2 — 규제 심사 설득력:** 비기술 심사위원이 "이 기술이 실제로 동작할 수 있다"고 납득할 수 있는 시각적 완성도
3. **G3 — 원클릭 실행:** 대표님의 Windows i5 PC에서 `python main.py` 또는 batch 파일 하나로 즉시 실행
4. **G4 — 성능 지표 시각화:** 추론 지연시간, FPS, 정확도 등 신청서 목표치를 실시간으로 화면에 표시
5. **G5 — 백엔드 연동:** 이벤트 발생 시 SafeWay Kids 백엔드로 전송, 학부모 앱 푸시 알림 시뮬레이션
6. **G6 — 안정성:** 데모 시연 중 크래시 없이 최소 10분 이상 연속 가동
7. **G7 — 4개 도메인 매핑:** 각 시나리오가 어느 하드웨어 도메인에 해당하는지 UI에서 명확히 표시

### Non-goals (이 PoC 범위 밖)

1. **NG1 — 실제 하드웨어 통합:** NVIDIA Jetson, LiDAR, 초음파 센서, CCTV 카메라 등 실제 하드웨어 연결 안 함
2. **NG2 — 프로덕션 모델 학습:** 커스텀 데이터셋으로 모델 학습/파인튜닝 안 함 (pretrained 모델 사용)
3. **NG3 — 다중 카메라:** 1대의 웹캠만 사용, 멀티 카메라 구성 안 함
4. **NG4 — GPU 필수 환경:** CUDA/GPU 없이 CPU만으로 동작해야 함
5. **NG5 — 앱스토어 배포:** 모바일 앱 빌드/배포 안 함 (기존 앱의 알림 수신만 확인)
6. **NG6 — 실시간 차량 추적:** GPS 기반 실시간 차량 위치 추적은 본 PoC 범위 밖
7. **NG7 — 멀티 유저 동시 접속:** 1명의 데모 진행자가 단일 브라우저에서 시연
8. **NG8 — 실제 센서 데이터 수집:** LiDAR/초음파는 전부 소프트웨어 시뮬레이션

## 5. Acceptance Criteria

### 시나리오별 수용 기준

#### AC1 — 안면 인식 승하차 (시나리오 ①)
- [ ] 웹캠 앞에 서면 2초 이내에 얼굴 감지 바운딩 박스 표시
- [ ] 등록된 얼굴을 95% 이상 정확도로 인식 (정면, 적정 조명)
- [ ] 인식 성공 시 "OOO 원생 탑승 확인" 메시지 + 초록색 바운딩 박스
- [ ] 미등록 얼굴은 "미등록" 표시 + 노란색 바운딩 박스
- [ ] 데모 중 실시간으로 새 얼굴 등록 가능 (이름 입력 + 캡처)
- [ ] 이벤트가 백엔드에 전송되고 로그에 기록됨

#### AC2 — 운행 중 이상 행동 감지 (시나리오 ②)
- [ ] 웹캠 앞에서 일어서는 동작 시 2초 이내 감지
- [ ] 감지 시 빨간색 바운딩 박스 + "위험: 이동 금지" 경고 표시
- [ ] 화면 하단에 경고 배너 표시
- [ ] 정상 착석 상태는 초록색 바운딩 박스 + "Normal" 표시
- [ ] 이벤트 로그에 이상 행동 기록

#### AC3 — 시동 OFF 잔류 인원 감지 (시나리오 ③)
- [ ] "시동 OFF" 버튼 클릭 후 잔류 인원 스캔 시작
- [ ] 사람 감지 시 빨간색 바운딩 박스 + "잔류 인원 감지" 경고
- [ ] 음성 또는 시각적 경고 + 관리자 경보 시뮬레이션
- [ ] 사람 없으면 "ALL CLEAR" 표시 (초록색)
- [ ] 이벤트가 백엔드에 전송됨

#### AC4 — 사각지대 어린이 감지 (시나리오 ④)
- [ ] 차량 Bird's Eye View UI 표시 (차량 + 감지 범위)
- [ ] 가상 어린이 객체가 접근 시 거리별 3단계 경고 (주의/경고/위험)
- [ ] LiDAR 포인트 클라우드 시각화 (방사형 레이 + 반사점)
- [ ] 초음파 거리 게이지 실시간 표시
- [ ] 위험 단계 시 "차량 이동 금지" 경고 + 경보음
- [ ] 시나리오 자동 재생 모드 지원

#### AC5 — 엣지 디바이스 시뮬레이션
- [ ] CPU 사용률, 메모리, 추론 지연시간 실시간 표시
- [ ] FPS 카운터 표시
- [ ] 모든 시나리오에서 5 FPS 이상 유지

#### AC6 — 통합 요건
- [ ] Windows i5 PC에서 원클릭 실행 (batch 파일 또는 단일 명령)
- [ ] 10분 이상 연속 가동 시 크래시 없음
- [ ] 한국어 UI (기술 용어 영문 병기)
- [ ] 4개 시나리오 간 전환이 2초 이내
- [ ] 이벤트 로그 패널에 모든 감지 이벤트 시간순 표시

## 6. 전문가 팀 구성 제안

본 PoC의 4개 도메인을 커버하기 위해 다음 4명의 Senior 전문가 리뷰어를 제안한다.

### R1 — Computer Vision / Edge AI 전문가

**역할:** 엣지 컴퓨팅 + 영상 분석 도메인 리드 리뷰어
**담당 도메인:** 도메인 1 (엣지 컴퓨팅), 도메인 2 (CCTV)
**검토 항목:**
- YOLOv8n CPU 추론 성능 최적화 전략
- MediaPipe Pose/Face 파이프라인 설계
- 이상 행동 감지 알고리즘 정확도 및 개선 방안
- 실시간 영상 처리 파이프라인 아키텍처
- i5 CPU 환경에서의 리소스 최적화
- 추론 지연시간 200ms 이하 달성 가능성

**필요 역량:** OpenCV, YOLO, MediaPipe, 엣지 추론 최적화, ONNX Runtime

### R2 — 안면 인식 / 생체 인증 전문가

**역할:** 안면 인식 터미널 도메인 리드 리뷰어
**담당 도메인:** 도메인 3 (안면 인식)
**검토 항목:**
- 안면 인식 라이브러리 선택 (face_recognition vs InsightFace vs 커스텀)
- CPU 환경에서 95% 인식률 달성 전략
- 마스크 착용 시 인식률 개선 방안
- 등록/인식 파이프라인 UX
- 조명 변화 대응 전략
- dlib 빌드 이슈 우회 방안 (Windows)
- Anti-spoofing (위조 방지) 기초 방안

**필요 역량:** face_recognition, InsightFace/ArcFace, dlib, 얼굴 임베딩, Liveness detection

### R3 — 센서 시뮬레이션 / 로보틱스 전문가

**역할:** 사각지대 감지 센서 도메인 리드 리뷰어
**담당 도메인:** 도메인 4 (사각지대 감지)
**검토 항목:**
- LiDAR 포인트 클라우드 시뮬레이션 정확성
- 초음파 센서 모델링 및 거리 계산
- 센서 융합 알고리즘 설계
- Bird's Eye View 시각화 UX
- 시나리오 자동 재생 로직
- 실제 센서 사양과의 gap 분석
- 규제 심사위원 설득력 평가

**필요 역량:** LiDAR 데이터 처리, 센서 융합, 로보틱스 시뮬레이션, Canvas/WebGL

### R4 — 풀스택 / 데모 UX 전문가

**역할:** 통합 데모 UX 및 시스템 아키텍처 리뷰어
**담당 도메인:** 전체 통합, UI/UX, 백엔드 연동
**검토 항목:**
- 4개 시나리오 통합 UI 설계 (단일 페이지 vs 탭 구조)
- 웹 프레임워크 선택 (Flask MJPEG vs Gradio vs React SPA)
- 데모 시나리오 흐름 UX (심사위원이 따라가기 쉬운 구조)
- 원클릭 실행 스크립트 설계
- 백엔드 이벤트 전송 + 모바일 푸시 연동
- 에러 복구 및 데모 안정성
- 시각적 완성도 (규제 심사 수준)
- 한국어 UI + 접근성

**필요 역량:** Flask/FastAPI, React/Canvas, 데모 UX 설계, 시스템 통합

## 8. 기존 코드 분석 및 Gap 식별

### 현재 구현 상태 (edge_ai/)

| 모듈 | 파일 | 현재 상태 | PoC 요구 대비 Gap |
|------|------|----------|-----------------|
| 메인 앱 | `main.py` | Flask+MJPEG, 3 모드 | 시나리오 ④ 미구현, UI 단순 |
| 안면 인식 | `face_manager.py` | MediaPipe FaceDetector + 히스토그램 매칭 | 95% 정확도 미달 가능, 마스크 대응 없음 |
| 이상 행동 | `behavior_detector.py` | YOLOv8n + MediaPipe Pose | 기본 동작하나 정확도 튜닝 필요 |
| 잔류 인원 | `passenger_detector.py` | YOLOv8n person detection | 기본 동작, multi-frame 분석 있음 |
| 이벤트 전송 | `event_sender.py` | REST POST to backend | 기본 동작 |
| 설정 | `config.py` | 환경변수 기반 | 도메인 4 설정 없음 |
| UI | `main.py` 내 HTML | 단순 MJPEG + 이벤트 로그 | 시각적 완성도 부족, 성능 대시보드 없음 |

### 주요 Gap

1. **시나리오 ④ 전체 구현 필요** — 사각지대 감지는 코드 없음
2. **안면 인식 정확도** — 히스토그램 매칭은 95% 달성 어려움, deep embedding 필요
3. **UI 시각적 완성도** — 현재 UI는 개발자용, 심사위원용으로 리디자인 필요
4. **성능 대시보드** — CPU/FPS/Latency 실시간 표시 없음
5. **한국어 UI** — 현재 영문 위주
6. **음성 경고** — 미구현
7. **야간 모드** — 미구현
8. **도메인 매핑 표시** — 각 시나리오가 어느 하드웨어에 대응하는지 UI 표시 없음
