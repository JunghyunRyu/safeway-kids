# Phase 0 — Intake: SAFEWAY KIDS 프로덕션 배포 준비

**작성일:** 2026-03-20
**요청자:** 프로젝트 책임자
**범위:** 기존 소프트웨어의 프로덕션 배포 준비 (코드 영역)

---

## 1. Requirement Brief (요구사항 개요)

### 배경
SAFEWAY KIDS 플랫폼은 백엔드(FastAPI), 모바일(Expo/React Native), 웹 대시보드(React), 랜딩 사이트(React)로 구성되며 개발 환경에서 동작하는 상태이다. 규제 샌드박스 신청은 완료되었으나, 심사 대비 실동작 데모 및 실제 서비스 배포를 위한 프로덕션 준비가 필요하다.

### 현재 상태
- 백엔드: 5,559 LOC, 28+ API, 80/82 테스트 통과
- 모바일: 6,472 LOC, 17개 화면, TypeScript 0 오류, 테스트 0개
- 웹: 1,093 LOC, 6페이지, TypeScript 0 오류, 테스트 0개
- 랜딩: 736 LOC, 3페이지, 완성
- 인프라: Docker Compose만 존재 (프로덕션 미구축)

### 요청 사항
기존 소프트웨어를 프로덕션 수준으로 끌어올려 실서비스 배포가 가능한 상태로 만들 것.

## 2. Goals / Non-goals

### Goals (목표)
1. **G-PROD-1:** 프로덕션 배포 가능한 인프라 코드 작성 (Docker, K8s 매니페스트, CI/CD)
2. **G-PROD-2:** 결제 시스템 실동작 준비 (PG SDK 연동 코드)
3. **G-PROD-3:** 배차 엔진 실거리 정확도 향상 (Kakao Mobility API 연동)
4. **G-PROD-4:** 프론트엔드 테스트 커버리지 확보 (모바일 + 웹)
5. **G-PROD-5:** 백엔드 테스트 안정화 (2개 오류 수정)
6. **G-PROD-6:** 미완성 기능 보완 (학생 역할, 관리자 학생화면, 엑셀 업로드)
7. **G-PROD-7:** 운영 도구 완성 (법적 증빙 아카이브, 관제 맵, 통계/리포트)
8. **G-PROD-8:** 에러 핸들링 및 UX 안정성 강화
9. **G-PROD-9:** SMS/FCM 프로덕션 전환 코드 준비

### Non-goals (비목표)
- NG-1: 엣지 AI 개발 (하드웨어 미보유)
- NG-2: 안면인식/이상행동 감지 구현
- NG-3: 실제 PG 가맹점 계약 체결 (사업 영역)
- NG-4: 앱스토어/플레이스토어 제출 (이번 범위 외)
- NG-5: 실서버 K8s 클러스터 프로비저닝 (코드만 작성)
- NG-6: 제3자 보안 감사 실시
- NG-7: 부하 테스트 실행 (인프라 미구축)

## 5. Acceptance Criteria (수용 기준) 초안

- [ ] AC-3: 웹 대시보드 핵심 페이지 테스트 존재 (최소 5개 테스트)
- [ ] AC-4: PG 결제 연동 코드 완성 (테스트 모드 동작)
- [ ] AC-5: Kakao Mobility 실거리 API 연동
- [ ] AC-6: Docker 프로덕션 이미지 빌드 성공
