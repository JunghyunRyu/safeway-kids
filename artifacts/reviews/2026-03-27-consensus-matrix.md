## 2. 합의 도출: Open Questions 해소

### Q1. 안면인식 라이브러리 선택 → **InsightFace ArcFace ONNX 확정**

| 리뷰어 | 의견 | 합의 |
|--------|------|------|
| R2 | InsightFace ArcFace ONNX 1순위 추천. dlib 배제. facenet-pytorch fallback | **채택** |
| R4 | dlib 시도 금지에 동의. InsightFace 권장 | **동의** |
| R1 | ONNX Runtime 기본 채택과 일치 | **호환** |

**결정:** `insightface` 패키지 직접 의존 대신, ArcFace ONNX 모델을 `onnxruntime`으로 직접 로드하는 경량 래퍼 구현. Fallback: facenet-pytorch.
**이유:** Windows pip install 한 줄, 빌드 이슈 없음, LFW 99.83%, CPU 50~100ms/face.

### Q2. 사각지대 시뮬레이션 아키텍처 → **프론트엔드(JS) 단독 처리**

| 리뷰어 | 의견 | 합의 |
|--------|------|------|
| R3 | 프론트엔드 JS 단독 처리, 이벤트만 백엔드 POST | **채택** |
| R4 | Canvas 2D 시각화에 동의 | **동의** |

**결정:** 사각지대 시뮬레이션 엔진과 렌더링은 JavaScript (Canvas 2D)로 구현. 경고 이벤트 발생 시에만 Flask 백엔드에 POST.
**이유:** Python 백엔드에 의존하지 않아 성능 격리 가능. 실시간 애니메이션에 적합.

### Q3. 수동 vs 자동 시나리오 → **둘 다 지원**

| 리뷰어 | 의견 | 합의 |
|--------|------|------|
| R3 | 자동 재생 기본 + 마우스 인터랙션 옵션 | **채택** |
| R4 | 데모 안정성을 위해 자동 재생 필수 | **동의** |

**결정:** 사전 정의 시나리오 자동 재생 (데모 기본 모드) + 마우스 드래그로 어린이 객체 이동 (인터랙티브 모드).

### Q4. UI 프레임워크 → **Flask + Socket.IO + Vanilla JS/Canvas**

| 리뷰어 | 의견 | 합의 |
|--------|------|------|
| R4 | Flask + Socket.IO + 정적 프론트엔드 최종 추천 | **채택** |
| R1 | WebSocket 기반 스트리밍 전환 권장 (MJPEG 탈피) | **호환** |
| R3 | Canvas 2D 렌더링에 동의 | **동의** |

**결정:** Flask-SocketIO 서버 + Vanilla JS + Canvas 2D. 영상 프레임은 WebSocket으로 base64 전송 (MJPEG 탈피). 사각지대는 Canvas 별도 렌더링.
**이유:** Python 단독으로 완결, Node.js 설치 불필요, 원클릭 실행 가능, Canvas로 오버레이 자유도 확보.

### Q5. 야간 모드 → **Nice-to-have (Phase 2 scope)**

**결정:** 기본 데모 범위에서 제외. 시간 여유 시 그레이스케일 + CLAHE 필터로 간단 구현.

### Q6. 음성 경고 → **Web Audio API + 사전 녹음 MP3**

**결정:** 한국어 음성 경고 MP3 파일 사전 제작. Web Audio API로 브라우저에서 재생. TTS는 불필요.

### Q7. 백엔드 연동 → **Standalone 기본 + 백엔드 자동 감지**

| 리뷰어 | 의견 | 합의 |
|--------|------|------|
| R4 | Standalone 기본, 백엔드 자동 감지 옵션 | **채택** |
| R1 | 독립 실행 모드 필요에 동의 | **동의** |

**결정:** 앱 시작 시 백엔드(localhost:8000) 헬스체크 → 응답 시 연동 모드, 미응답 시 standalone 모드 자동 전환. Standalone에서도 모든 UI 기능 동작 (이벤트 로그만 로컬 저장).

## 4. 불일치 해소

### 4.1 R3의 경고 거리 vs 가안의 경고 거리

- **가안:** 주의 5m / 경고 3m / 위험 1.5m
- **R3 지적:** 초음파 센서 범위가 4m인데 5m 주의 경고는 불가
- **해소:** 경고 거리를 센서 범위 내로 조정:
  - LiDAR 경고: 주의 10m / 경고 5m / 위험 2m (LiDAR 범위 100m)
  - 초음파 경고: 주의 3.5m / 경고 2m / 위험 0.8m (초음파 범위 4m)
  - **융합 경고:** 주의 4m / 경고 2.5m / 위험 1.0m (두 센서 합산)

### 4.2 R1의 FPS 목표 vs 가안의 5 FPS

- **가안:** 5 FPS 이상
- **R1 지적:** 추론 200ms + 오버헤드 = 실질 3~4 FPS
- **해소:** 목표를 "3 FPS 이상 (체감 5 FPS)" 로 조정. 프레임 스킵으로 최신 추론 결과를 이전 프레임에 오버레이하여 체감 FPS 향상.

### 4.3 R4의 MJPEG vs R1의 WebSocket

- **합의:** MJPEG 폐기, WebSocket(Socket.IO)으로 base64 JPEG 프레임 전송. R1과 R4 의견 일치.
