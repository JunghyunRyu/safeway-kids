# 운전기사 페르소나 프로덕트 리뷰

**날짜**: 2026-03-21
**리뷰 대상**: 기사용 모바일 앱 (`mobile/src/screens/driver/*`, 관련 hooks/API)
**리뷰어**: 기사A (이상철, 55세), 기사B (김영수, 48세), 기사C (박미영, 45세)

---

## 1. 기사A — 이상철 (55세, 전세버스 15년, 스마트폰 미숙)

### 전체 인상
> "솔직히 처음 열었을 때 뭘 눌러야 하는지 모르겠어요. 기존에는 원장님한테 전화 받으면 '오늘 몇 시에 어디 가' 하고 끝이었는데, 이거는 화면이 여러 개라 헷갈립니다."

### 시나리오별 피드백

#### 출근 → 배차 확인 (HomeScreen)
- **좋은 점**: 차량 번호, 탑승 인원이 한눈에 보임. 숫자가 크게 나와서 좋음.
- **문제점**:
  - "오늘 운행 요약"이라고만 쓰여있고, **내가 지금 뭘 해야 하는지** 안내가 없음. "출발하세요" 같은 명확한 액션 유도가 필요함.
  - **글씨가 작음**. `Typography.sizes.sm`(12px), `base`(14px) 수준은 55세 눈에 잘 안 보임. 최소 16px 이상이어야 함.
  - 안전도우미 이름은 나오는데 **전화번호가 없음**. 처음 만나는 도우미인데 어떻게 연락하나?

#### 운전 중 네비게이션 (MapScreen)
- **심각한 문제**: 지도에 정류장 마커만 표시되고, **카카오네비나 T맵으로 연동하는 기능이 없음**. 나는 네비 없이는 못 다님. 마커 눌러서 네비 앱으로 바로 넘어가야 함.
- GPS 상태 표시(초록 점)는 너무 작아서 운전 중 안 보임. 신호등 크기는 돼야지.
- **운전 중에 WebView 지도를 조작하는 건 위험함**. 지도보다는 "다음 정류장: OO아파트 앞, 3분 후 도착" 같은 **텍스트 기반 안내**가 훨씬 나음.

#### 정류장 도착 → 탑승 체크 (RouteScreen)
- **탑승 버튼이 너무 작음**. `minHeight: 40` + `paddingVertical: Spacing.sm(8px)`인데, 운전석에서 한 손으로 누르기엔 절대 부족함. **최소 56px 높이, 전체 너비**로 만들어야 함.
- "탑승" 두 글자만 있으면 뭔 뜻인지 모름. **"김민수 탑승 확인"** 처럼 학생 이름이 버튼에 들어가야 함.
- **탑승/하차 실수 취소 기능이 없음**. 잘못 눌렀을 때 되돌릴 수가 없음. 확인 팝업도 없이 바로 처리됨.
- 카드 형태로 스크롤해야 하는데, 15명이면 스크롤이 길어짐. **전체 일괄 탑승 버튼**이 필요함 (같은 정류장에서 5명 동시 탑승할 때).

#### 긴급 상황
- **긴급 버튼(SOS)이 아예 없음**. 이게 제일 큰 문제임. 사고 나면 누구한테 전화해야 하는지, 앱에서 바로 신고할 수 있어야 함.
- 학생이 안 나오면? "미탑승" 처리 버튼이 없음. 현재는 그냥 체크 안 하고 넘어가는 수밖에 없는데, 학부모/학원에 자동 알림이 가야 함.
- 차량 고장 시 본사/운수업체에 알리는 기능 없음.

#### 퇴근 → 일일 리포트
- **일일 리포트 화면 자체가 없음**. 오늘 몇 명 태웠고, 몇 km 운행했고, 특이사항 뭐였는지 정리해주는 화면이 필요함.
- 운행 종료 버튼이 없어서 GPS가 언제 꺼지는지 모름.

### 기사A 종합 점수: **40/100**
> "전화로 하던 게 솔직히 더 편해요. 이 앱 쓰려면 교육 한 번 받아야 할 것 같은데, 그래도 버튼이 이렇게 작으면 소용없어요."

## 2. 기사B — 김영수 (48세, 택시 10년, 네비 익숙, 효율 중시)

### 전체 인상
> "구조는 괜찮은데, 실제 운행 효율을 높여주는 기능이 부족합니다. 택시 앱들은 다음 콜이 딱딱 뜨는데, 이건 제가 직접 찾아봐야 해요."

### 시나리오별 피드백

#### 출근 → 배차 확인 (HomeScreen)
- **좋은 점**: 탑승/총학생/완료 통계 카드가 깔끔함. 진행 상황 한눈에 파악 가능.
- **개선점**:
  - **첫 정류장까지의 예상 소요시간이 없음**. 몇 시에 출발해야 하는지 계산을 내가 해야 함.
  - 학원별로 분리된 스케줄이면 좋겠음. 하루 2개 학원인데 어느 학원 것인지 한눈에 구분 안 됨.

#### 운전 중 네비게이션 (MapScreen)
- **AI 최적화 노선이라고 써있는데** (`v{routePlan.version} · {routePlan.total_distance_km}km`), 실제로 **턴바이턴 네비게이션이 아님**. 그냥 마커만 찍혀 있음.
- **핵심 요구사항**: "노선" 탭에서 정류장 터치 → 카카오네비/T맵 자동 실행. 이것만 되면 80% 해결됨.
- 경로 표시(`polyline`)가 지도에 그려지지 않음. 최적화 노선이라면서 경로선이 없으면 의미가 없음.
- 교통 상황 반영이 없음. 택시 할 때는 실시간 교통이 생명이었는데.

#### 정류장 도착 → 탑승/하차 체크 (RouteScreen)
- **탑승과 하차가 같은 리스트에 섞여 있음**. 픽업 루트와 하차 루트를 분리해야 함. 아침에는 집→학원(탑승 체크), 오후에는 학원→집(하차 체크)인데 이 구분이 없음.
- 정류장 순번 원(indexCircle, 36x36)은 적절한 크기.
- **다음 정류장 하이라이트가 없음**. 10명 중 5명 완료했으면, 6번째가 강조되어야 하는데 전부 같은 스타일임.
- `Pressable` 버튼에 `hitSlop`이 설정 안 됨 (코드에서 확인). 터치 영역이 시각적 크기와 동일하면 운전 중 미스터치 많음.

#### 효율 관련
- **동선 최적화 결과를 검증할 수 없음**. "AI 최적화"라고 하는데 내가 보기엔 비효율적일 수 있음. 수동으로 순서 변경할 수 있어야 함.
- **예상 도착시간(ETA) 표시가 없음**. 지도 HTML에 `eta-overlay` div가 있는데 (`display: none`) 실제로 사용되지 않음. 구현 안 된 것 같음.
- 운행 시간 기록이 없음. 첫 탑승부터 마지막 하차까지 총 운행시간을 알고 싶음.

#### 긴급 상황
- 기사A와 동감. **SOS 버튼 필수**.
- 추가로 **학생 미탑승 시 자동 대기 타이머**가 있으면 좋겠음. "3분 대기 후 다음 정류장으로 이동" 같은 프로세스.

### 기사B 종합 점수: **55/100**
> "뼈대는 괜찮습니다. 근데 실제 운행하는 기사 입장을 모르고 만든 느낌이에요. 네비 연동이랑 노선 순서 조절만 되면 쓸 만할 것 같습니다."

## 3. 기사C — 박미영 (45세, 스쿨버스 3년, 안전 최우선, 학부모 소통 중시)

### 전체 인상
> "아이들 안전에 관련된 기능이 너무 없어요. 탑승 확인만 있고, 그 외 안전 관련 기능은 하나도 없습니다. 학부모님들한테 메시지 보내는 것도 안 되고요."

### 시나리오별 피드백

#### 출근 → 배차 확인 (HomeScreen)
- **좋은 점**: 안전도우미 이름 표시는 좋음. 같이 일하는 분이 누군지 아는 게 중요함.
- **문제점**:
  - **학생별 특이사항/알레르기/비상연락처가 안 보임**. 스쿨버스에서 이게 제일 중요함. "김민수 - 차멀미 심함, 앞자리 배정" 같은 정보가 학생 카드에 표시되어야 함.
  - **출석부 기능이 없음**. 사진 대조로 본인 확인하거나, 최소한 학생 사진이 표시되어야 함.

#### 학생 탑승 체크 (RouteScreen)
- **"탑승" 버튼 하나로 끝나면 안 됨**. 실제로는:
  1. 학생 본인 확인 (얼굴/이름)
  2. 안전벨트 착용 확인
  3. 탑승 보호자 확인 (초등 저학년)

  이 과정이 앱에 반영되어야 함. 최소한 **체크리스트** 형태.
- **탑승 시 학부모에게 자동 알림**이 가는지 확인 안 됨. 코드상 `markBoarded` API만 호출하고, 푸시 알림이 학부모에게 가는지는 백엔드 로직에 달려있음. **기사 화면에서 "학부모 알림 전송됨" 확인 표시가 필요함**.
- **하차 시에도 마찬가지**. "하차 완료 → 학부모 알림" 확인이 기사 화면에 표시되어야 안심됨.

#### 학생 하차 체크
- **하차 시 인수자 확인 기능이 없음**. 초등 저학년은 보호자 인수가 원칙인데, 누가 데려갔는지 기록 안 됨.
- **차량 내 잔류 학생 확인 기능 없음**. 하차 완료 후 "차량 내 학생 없음 확인" 버튼이 있어야 함. 스쿨버스 사고 중 잠든 아이 방치가 가장 심각한 건인데, 이 체크가 빠져있음.
- 모든 학생 하차 완료 시 **자동 운행 종료 알림**이 없음.

#### 학부모 소통
- **기사→학부모 메시지 기능이 아예 없음**. "교통 체증으로 10분 지연됩니다" 같은 안내를 보낼 방법이 없음.
- 현재는 카톡 단체방을 따로 쓰는데, 앱 내에서 통합되면 훨씬 좋겠음.
- 학부모가 "오늘 아이 결석" 알림을 보냈을 때 기사 화면에 실시간으로 반영되는지? 현재 `useFocusEffect`로 화면 포커스 시에만 데이터를 갱신함. **실시간 푸시 반영 안 됨**.

#### 긴급 상황
- **가장 심각한 부재 기능**:
  1. **SOS/긴급 호출 버튼** — 사고 시 112/119 + 본사 + 학부모 동시 알림
  2. **학생 건강 이상 신고** — 아이가 아프거나 다쳤을 때 기록 + 학부모 알림
  3. **차량 내 CCTV 연동** — 차 안 상황 확인 (향후)
  4. **비상 정차 기록** — 어디서 왜 정차했는지 로그
- 현재 앱에는 위 기능 **모두 없음**. 긴급 상황 시 앱은 쓸모없고 개인 전화기로 처리해야 함.

#### 퇴근 → 리포트
- 기사A와 동감. **일일 운행 보고서** 필요.
- 추가로 **특이사항 메모 기능** 필요. "김민수 오늘 차멀미 호소", "박지원 하교 시 할머니가 데리러 옴" 같은 기록.

### 기사C 종합 점수: **35/100**
> "안전을 최우선으로 만들었다면서, 정작 안전 관련 기능이 탑승 체크밖에 없어요. 학부모님이 이 앱 보시면 '아이를 이 앱에 맡길 수 있나' 하실 거예요."

## 기사 3인 공통 의견

1. **"안전이 이름인 서비스인데, 긴급 상황 대응이 제로입니다."** — SOS 버튼은 반드시 홈 화면 상단 고정 또는 모든 화면에서 접근 가능해야 함.

2. **"네비 연동 없이는 못 씁니다."** — 지도에 마커만 찍어놓는 건 기사에게 의미 없음. 원터치로 네비 앱 실행이 핵심.

3. **"운전 중에 쓸 수 있는 앱이 아닙니다."** — 버튼이 작고, 확인 없이 처리되고, 음성 지원이 없음. 운전대 잡고 한 손으로 조작하는 상황을 전혀 고려하지 않은 UI.

4. **"카톡 단체방을 대체하지 못합니다."** — 학부모와의 소통 채널이 앱에 없어서 결국 카톡을 병행해야 함. 이러면 앱을 쓸 이유가 줄어듦.

5. **"하루 끝나고 뭘 했는지 정리가 안 됩니다."** — 일일 보고서/운행 기록 부재로, 사고 시 증빙 자료로도 쓸 수 없음.
