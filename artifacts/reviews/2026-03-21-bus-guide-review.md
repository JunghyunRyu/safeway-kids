## 1. 가이드A (정수빈) — 하루 일과 시뮬레이션

### 출근~퇴근 시뮬레이션

**06:30 출근**
- 앱에 `safety_escort` 역할로 로그인. "내 근무" 탭(`ShiftsScreen`)에서 오늘 배정된 근무 확인.
- 출근 버튼 누름 → check-in 처리됨. 여기까지는 OK.
- **문제 시작:** 출근 후 "오늘 내가 탈 차량 번호가 뭐지?" → 근무 카드에 차량 정보가 없음. `EscortShift` 모델에는 `vehicle_assignment_id`만 있고, 화면에는 날짜/수당/출퇴근 시간만 표시됨. 차량 번호판, 기사 이름, 어떤 학원 루트인지 전혀 안 보임.
- **더 큰 문제:** "오늘 태울 아이들이 누구지?" → 안전도우미 앱에는 학생 목록 화면이 아예 없음. `EscortTabNavigator`에는 근무/가용시간/지도/프로필 4개 탭만 있고, 기사 앱의 `RouteScreen`(정류장 목록)에 해당하는 화면이 없음.

**07:00 첫 루트 출발**
- 기사가 운전하고 나는 뒤에서 아이들 관리. 기사 앱(`RouteScreen`)에는 학생 이름/학원/픽업시간/탑승 버튼이 있음.
- 그런데 나(가이드)는 이 화면을 볼 수 없음. 안전도우미 계정으로는 `DriverRouteScreen`에 접근 불가 — 네비게이터 구조상 escort 탭에만 접근 가능.
- **현실:** 기사한테 "다음 정류장에서 누가 타요?"라고 매번 물어봐야 함. 기사는 운전 중이라 대답하기 어려움.

**07:15 첫 정류장 도착**
- 아이가 올라탐. 나는 안전벨트 채워주고 자리 안내해야 함.
- **탑승 체크:** `mark_boarded` API는 `DRIVER`와 `SAFETY_ESCORT` 둘 다 호출 가능함 (라우터에서 `require_roles(UserRole.DRIVER, UserRole.SAFETY_ESCORT)` 확인). 백엔드는 OK. **하지만** 안전도우미 앱 UI에는 탑승/하차 버튼이 있는 화면 자체가 없음. API 권한은 있으나 UI가 없는 상태.
- **아이 확인:** "이 아이가 맞나?" → 학생 사진이 없음. `DriverDailySchedule` 인터페이스에 `student_name`만 있고 `profile_photo_url`이 없음. 백엔드 `Student` 모델에는 `profile_photo_url` 필드가 있지만 스케줄 API 응답에 포함되지 않음.
- **특이사항:** "이 아이는 알레르기가 있나? 차멀미는?" → 학생 모델에 알레르기, 특이사항, 비상연락처 필드가 아예 없음. `Student` 모델에는 `name`, `date_of_birth`, `grade`, `profile_photo_url`, `is_active`만 있음.

**07:30~08:30 루트 순회**
- 15~20개 정류장을 돌면서 아이들 태우고 내림.
- **오늘 결석/취소 아이:** 기사 앱에서는 `cancelled` 상태가 회색으로 보이지만, 가이드에게는 이 정보가 안 보임. 정류장에 도착했는데 아이가 안 나오면 기사한테 물어봐야 함.
- **좌석 배정:** 차멀미 아이 앞자리 등 좌석 배정 정보가 시스템에 없음. 좌석 관련 모델/API 없음.
- **안전벨트 체크:** 안전벨트 체크 기록 기능 없음. 모델에도 API에도 없음.

**08:30 학원 도착, 하차**
- 아이들 하차시키고 인원 확인.
- **잔류 아이 확인:** SRS에는 "잔류 인원 감지"가 핵심 기능으로 명시되어 있지만 (6.1절), 이건 엣지 AI(NVIDIA Jetson) 하드웨어 기능이라 앱에는 없음. 가이드가 수동으로 차 안 확인하고 "차량 내 잔류 아이 없음" 확인 버튼 같은 건 없음.
- **학원 도착 알림:** 가이드가 학원에 "도착했어요" 보내는 기능 없음. 기사의 하차 처리 시 학부모에게 알림은 감(`send_alighting_notification`) 하지만 학원 측에 알림은 없음.

**09:00~14:00 대기 시간**
- 오전 루트 끝. 오후 루트까지 대기.
- 가용시간 관리(`AvailabilityScreen`)로 내일 일정 등록 가능. 이 기능은 잘 되어 있음.

**14:00~16:00 오후 루트 (학원→집)**
- 동일한 문제 반복. 학생 목록 없음, 사진 없음, 특이사항 없음.
- **아이가 울거나 다침:** 즉시 보고 기능 없음. SOS/긴급 버튼 없음 (안전도우미 앱에). 기사 앱에도 SOS 버튼은 없음 — SRS 7.2절의 "비상 오버라이드 알람"은 엣지 디바이스 연동 기능이라 미구현.
- **기사와 소통:** 인앱 채팅/메시지 기능 없음. 같은 차 안에 있으니 말로 하면 되지만, 기사가 운전 중일 때는 위험.

**16:30 퇴근**
- "내 근무" 화면에서 퇴근 버튼 → check-out 처리.
- **오늘 특이사항 보고:** 운행 보고서 작성 기능 없음. "3번째 정류장에서 민서가 넘어져서 무릎 까졌음" 같은 걸 기록할 곳이 없음.

### 앱으로 해결되는 것 vs 안 되는 것

| 해결됨 | 해결 안 됨 |
|--------|-----------|
| 출퇴근 체크 (check-in/out) | 오늘 탑승 아이 목록 확인 |
| 가용시간 등록 | 학생 사진/얼굴 확인 |
| 지도에서 차량 위치 확인 | 학생 특이사항(알레르기, 비상연락처) |
| 수당 확인 | 탑승/하차 체크 (UI 없음, API만 있음) |
| | 좌석 배정 정보 |
| | 안전벨트 체크 기록 |
| | 잔류 아이 최종 확인 |
| | SOS/긴급 보고 |
| | 기사-가이드 인앱 소통 |
| | 운행 보고서/특이사항 기록 |
| | 학원 도착 알림 발송 |
| | 결석/취소 아이 실시간 확인 |

### 가장 필요한 기능 TOP 5

1. **오늘 탑승 학생 목록 + 사진 + 특이사항** — 가장 기본이고 가장 절실. 아이 이름만으로는 얼굴을 모를 때 확인 불가. 알레르기, 분리불안, 비상연락처 필수.
2. **탑승/하차 체크 UI (가이드용)** — API 권한은 이미 있으니 `RouteScreen`을 가이드 탭에도 연결하거나 가이드 전용 체크인 화면 필요.
3. **하차 후 차량 잔류 확인 버튼** — "차량 내 아이 없음 확인" 체크리스트. 세림이법 핵심 의무.
4. **SOS/긴급 보고 버튼** — 아이 부상, 차량 사고 시 원터치로 학원+학부모+관제센터 동시 알림.
5. **운행 특이사항 기록** — 종이 보고서 대체. 사진 첨부 가능하면 더 좋음.

### 안전 관련 빈틈

- **세림이법 준수 증명 불가:** 동승보호자가 탑승했다는 증거가 check-in 시간뿐. 실제로 차에 타고 있었는지, 안전벨트를 확인했는지, 잔류 확인을 했는지 기록 없음.
- **학생 인계 기록 공백:** 누가 아이를 인수했는지(학원 선생님? 부모?) 기록이 없음.
- **사고 발생 시 보고 체계 부재:** 앱에 사고/사건 보고 기능이 없어서 카톡으로 보고할 수밖에 없음 — 현재와 동일.

### 종이 출석부 대비 장단점

| 장점 | 단점 |
|------|------|
| (현재로선) 기사 앱의 디지털 탑승 기록이 자동 저장됨 | 가이드 본인은 디지털 체크를 못 함 (UI 없음) |
| GPS로 운행 기록 자동 생성 | 학생 사진이 없어서 종이 출석부에 붙인 사진보다 못함 |
| 학부모 자동 알림 | 종이에는 "차멀미 주의" 같은 메모를 자유롭게 적을 수 있는데 앱에는 불가 |
| 데이터 누적/분석 가능 | 비 오는 날 폰 꺼내기 더 불편 (종이는 코팅해서 씀) |

## 2. 가이드B (김태영) — 하루 일과 시뮬레이션

### 출근~퇴근 시뮬레이션

**07:00 출근 (첫 주)**
- 앱 설치하고 로그인. "내 근무"에 오늘 배정 뜸. 출근 버튼 누름.
- **"오늘 어디로 가야 하죠?"** → 근무 카드에 차량 번호, 기사 이름, 루트 정보가 없음. 학원에 전화해서 물어봐야 함.
- **교육 자료:** 앱에 신규 가이드 매뉴얼/체크리스트 같은 건 없음. 컴플라이언스 모듈에 `SAFETY_TRAINING` 문서 타입은 있지만 이건 관리자가 업로드하는 서류이지 가이드가 보는 교육 콘텐츠가 아님.

**07:15 차량 탑승**
- 기사님이 알려주는 대로 따라감.
- **"이 아이가 누구예요?"** → 20명 이름을 아직 다 못 외움. 앱에서 확인하고 싶은데 학생 목록이 없음.
- 수빈 언니(가이드A)는 1년 했으니 다 아는데, 나는 처음이라 사진이라도 있으면 좋겠음.
- **"이 아이는 몇 번 자리에 앉혀야 해요?"** → 좌석 배정 정보 없음. 기사님한테 물어봄 → 기사님도 "아무 데나 앉혀" 라고 함.

**07:30 정류장에서**
- 아이가 올라탐. 탑승 체크를 해야 하는데 내 폰에는 버튼이 없음.
- 기사님이 자기 폰으로 탑승 처리. 나는 안전벨트만 채워줌.
- **"안전벨트 다 채웠어요"를 기록할 곳이 없음.** 그냥 눈으로 확인하고 넘어감.

**08:00 아이가 울기 시작**
- 엄마 보고 싶다고 울음. 분리불안인지, 아픈 건지 모르겠음.
- **"이 아이 보호자 번호가 뭐예요?"** → 앱에 비상연락처 없음. 학원에 전화해서 물어봐야 함.
- **앱에 "이 아이가 울고 있어요" 보고 기능 없음.** 기사님한테 말하고 기사님이 학원에 전화.

**08:20 학원 도착, 하차**
- 아이들 내리고 인원 확인. "18명 탔는데 18명 내렸나?" 수동으로 머릿수 셈.
- 차 안 구석구석 확인 — 뒤 좌석 밑에 아이 있을 수 있으니. **이걸 했다는 기록을 남길 곳이 없음.**
- 학원 선생님한테 "아이들 다 인계했습니다" → 카톡으로 보고.

**14:00 오후 루트**
- 기사님이 바뀜 (오후에 다른 기사님이 옴). 나는 그대로인데 기사님이 바뀌었으니 "오늘 오후 특이사항" 인수인계가 필요한데 시스템에 그런 기능 없음.

**16:00 퇴근**
- 퇴근 버튼 누름. 수당 5만원 확인.

### 앱으로 해결되는 것 vs 안 되는 것

| 해결됨 | 해결 안 됨 |
|--------|-----------|
| 출퇴근 관리 | 아이 이름-얼굴 매칭 (사진 없음) |
| 수당 확인 | 배정 차량/기사/루트 정보 |
| 가용시간 등록 | 신규 가이드 교육 콘텐츠 |
| | 아이별 특이사항/비상연락처 |
| | 탑승/하차 체크 UI |
| | 안전벨트 확인 기록 |
| | 잔류 확인 체크리스트 |
| | 사건/사고 보고 |
| | 기사 인수인계 |

### 가장 필요한 기능 TOP 5

1. **학생 사진이 있는 탑승 명부** — 이름만으로는 아이를 못 알아봄. 사진, 학년, 어느 학원 아이인지 한눈에 보이는 카드형 UI 필요.
2. **오늘의 루트 정보** — 배정된 차량 번호, 기사 이름, 순회할 정류장 순서, 각 정류장에서 타는 아이 목록.
3. **비상연락처 즉시 조회** — 아이 카드 터치하면 보호자 전화번호 바로 나오기.
4. **안전 체크리스트** — 탑승 시 안전벨트 확인, 하차 시 잔류 확인, 각 단계별 체크 후 확인 버튼.
5. **간단한 메모/보고 기능** — "민서가 배 아프다고 함", "3번 정류장 앞 공사 중" 같은 메모를 남기면 학원과 다음 시프트 가이드가 볼 수 있는 기능.

### 안전 관련 빈틈

- **신규 가이드 온보딩 부재:** 매뉴얼이 앱에 없어서 기사한테 전적으로 의존. 기사가 안 알려주면 모르고 지나감.
- **아이 식별 불가:** 사진 없이 이름만으로 15~20명을 구분해야 함. 동명이인 있으면 큰 문제.
- **인계 기록 없음:** 아이를 학원에 넘길 때 "누구한테 넘겼다"는 기록이 없어서 책임 소재 불분명.

### 종이 출석부 대비 장단점

| 장점 | 단점 |
|------|------|
| 폰 하나면 되니 종이+펜 안 들고 다녀도 됨 | 종이에는 아이 사진 붙이고 메모 적을 수 있는데 앱은 안 됨 |
| 출퇴근이 자동 기록됨 | 종이 출석부는 한눈에 전체가 보이는데 앱은 스크롤해야 함 |
| 데이터 못 잃어버림 | 앱 로딩 시간 > 종이 펼치는 시간 |

## 3. 공통 검토 포인트별 분석

### 3.1 일상 운행

| 검토 항목 | 현재 상태 | 판정 |
|-----------|----------|------|
| 가이드 전용 앱/화면 존재 여부 | `EscortTabNavigator`로 별도 탭 구성 있음 (근무/가용시간/지도/프로필) | **부분 지원** — 존재하나 핵심 기능 부족 |
| 오늘 탈 아이 목록+사진 미리보기 | 안전도우미 앱에 학생 목록 화면 없음. 스케줄 API에 사진 미포함 | **미지원** |
| 탑승/하차 체크 가능 여부 | 백엔드 API 권한 있음 (`SAFETY_ESCORT` 허용). 프론트 UI 없음 | **API만 지원, UI 미지원** |

## 4. 핵심 발견 사항 요약

### 구조적 문제 (Critical)

1. **안전도우미 앱이 사실상 "출퇴근+수당 관리 앱"에 불과함.** 동승보호자의 핵심 업무인 '아이 관리'를 지원하는 기능이 전혀 없음. 탑승 명부, 학생 정보, 체크인/체크아웃 UI가 모두 부재.

2. **SRS에 명시된 "운전자 및 동승 보호자용 애플리케이션"이 사실상 "운전자 전용"으로 구현됨.** SRS 7.2절은 "Driver & Guardian App"으로 통합 앱을 명시했으나, 실제로는 기사 탭에만 학생 목록/탑승 관리가 있고 안전도우미 탭에는 없음.

3. **세림이법 준수 증빙 불가.** 동승보호자 탑승 의무는 check-in/out 시간으로 간접 증명 가능하나, 안전벨트 확인, 잔류 확인, 인계 기록 등 실질적 안전 활동의 증거가 없음.

### 데이터 모델 문제 (High)

4. **학생 특이사항 필드 부재.** `Student` 모델에 알레르기, 건강 특이사항, 비상연락처(보호자 외), 차멀미 여부 등의 필드가 없음. 아이 안전 관리의 기초 데이터가 빠져 있음.

5. **근무 배정에 차량/루트 정보 미연결 (UI 레벨).** `EscortShift`는 `vehicle_assignment_id`로 차량 배정과 연결되어 있지만, 프론트엔드의 `ShiftCard`에는 이 정보가 표시되지 않음.

### UX 문제 (Medium)

6. **가이드가 기사 앱의 핵심 화면을 공유받지 못함.** `RouteScreen`(정류장 목록+탑승 버튼), `HomeScreen`(오늘 요약)이 기사 전용. API 권한은 열려 있으나 네비게이터에서 연결 안 됨.

7. **사건/사고 보고 체계 부재.** 아이 부상, 차량 사고, 이상 상황 발생 시 앱 내 보고 기능이 없음. 현재와 같이 카톡으로 보고해야 함.

## 6. 결론

현재 SAFEWAY KIDS의 안전도우미 앱은 **인력 관리(출퇴근/수당/가용시간)** 측면에서는 기본 기능이 갖춰져 있으나, **현장 안전 관리(학생 확인/탑승 체크/안전 점검/사고 보고)** 측면에서는 거의 빈 상태입니다.

SRS에서 "동승 보호자용 앱"을 기사 앱과 통합 명세했음에도, 실제 구현에서 기사의 핵심 기능(학생 목록, 탑승/하차 처리)이 안전도우미 탭으로 연결되지 않아, 가이드는 사실상 **기사에게 구두로 의존하는 종이 출석부 시절과 동일한 상황**입니다.

가장 시급한 것은 **G1 (RouteScreen 공유)**로, 이것만 해도 가이드가 오늘의 학생 목록을 보고 탑승/하차 체크를 직접 할 수 있게 됩니다. 비용 대비 효과가 가장 큰 조치입니다.

> **정수빈(가이드A) 한마디:** "1년 동안 일하면서 아이들 이름은 다 외웠지만, 신규 가이드가 올 때마다 '이 아이 누구예요' 물어보는 게 반복돼요. 앱에 사진이랑 특이사항만 있어도 80%는 해결됩니다."

> **김태영(가이드B) 한마디:** "출퇴근 앱으로는 좋은데, 정작 아이들 관리할 때는 기사님한테 다 물어봐야 해서... 차라리 종이 출석부에 사진 붙인 게 더 편했어요."
