# 학부모 피드백

> 코드 리뷰 기반 UX 시뮬레이션. 모바일 앱 `mobile/src/screens/parent/` 전체 화면, 네비게이션, 알림 시스템, API 계층을 분석한 결과.

---

## 학부모A (김지영, 38세) 피드백

> 초등 2학년 딸, 태권도+영어학원 2곳 셔틀 이용. 스마트폰 익숙하나 앱에 까다로움. 아이 안전이 최우선.

### 좋은 점

1. **홈 화면이 깔끔하다.** 앱 켜자마자 "오늘 일정 2건"이 바로 보인다. 아이 이름 + 상태(예정/탑승 중/완료) + 픽업 시간이 카드 형태로 딱 나와서 한눈에 파악 가능. (`HomeScreen.tsx:57-71`)

2. **실시간 추적 지도가 있다.** 위치추적 탭에서 셔틀 버스 위치를 지도로 볼 수 있고, "연결됨" 상태가 초록색 점으로 표시돼서 실시간인지 아닌지 직관적으로 알 수 있다. (`MapScreen.tsx:153-179`)

3. **알림이 푸시+SMS 이중으로 온다.** 탑승 알림이 안 왔을 때 SMS 폴백이 있다는 건 안심이 된다. 앱 알림을 놓칠 수 있으니까. (`notification/service.py:14-31`)

4. **스케줄 취소 시 확인 다이얼로그가 뜬다.** 실수로 취소하는 걸 방지해줘서 좋다. (`ScheduleScreen.tsx:252-256`)

### 불편한 점

1. **홈 화면에서 아이를 탭해도 아무 반응이 없다.** 카드를 눌러서 상세 정보(어떤 학원, 어떤 차량, 기사 이름, 예상 도착 시간)를 보고 싶은데, ScheduleCard가 그냥 View라서 터치 이벤트가 없다. (`HomeScreen.tsx:57` — Pressable이 아닌 View로 렌더링) 딸 이름만 달랑 있고 어디로 가는 건지, 누가 운전하는 건지 안 보인다.

2. **학원별로 구분이 안 된다.** 태권도랑 영어학원 두 군데 다니는데, 스케줄 카드에 학원 이름이 없다. `DailySchedule` 타입에 `academy_id`는 있지만 화면에 학원 이름을 표시하지 않는다. (`ScheduleScreen.tsx:262-275` — studentName, pickupTime, status만 표시) 두 개가 비슷한 시간대면 어떤 게 태권도고 어떤 게 영어학원인지 구별이 안 된다.

3. **지도에서 내 아이 픽업 위치가 안 보인다.** 셔틀 버스 위치만 나오고, 우리 아이가 어디서 타는지(정류장 위치)가 지도에 안 찍힌다. `setStops` 메시지를 처리하는 코드는 mapHtml에 있지만 MapScreen에서 실제로 보내는 곳이 없다. 버스가 어디쯤 와 있는지 알아도 "우리 집 앞 정류장까지 몇 분인지"를 직관적으로 판단하기 어렵다.

4. **ETA가 "3정거장 전" 같은 형태인데, 시간이 아니다.** 몇 분 남았는지가 중요한데 정거장 수는 감이 안 온다. 정거장 간 거리가 다 다르니까. (`MapScreen.tsx:116-123`)

5. **스케줄에서 "새 스케줄 추가"가 없다.** 날짜를 넘겨가며 확인만 가능하고, 임시로 하루 더 태우고 싶을 때(예: 보충수업) 앱에서 추가할 수 없다. 학원에 전화해야 하나?

6. **프로필 화면에서 자녀 정보를 볼 수 없다.** 등록된 자녀 목록, 각 자녀의 학원 정보, 알레르기 같은 특이사항을 확인/수정할 곳이 없다. (`ProfileScreen.tsx` — 계정 정보와 앱 정보만 있음)

### 불안한 점

1. **하차 후 안전 확인이 없다.** "하차 완료" 알림은 오는데, 아이가 학원 건물 안까지 들어갔는지는 알 수 없다. 하차 지점과 학원 입구가 다를 수 있는데, 그 사이에서 무슨 일이 생기면? 안전도우미가 확인해주는지, 학원 쪽에서 출석 확인을 해주는지 앱에서 알 수가 없다.

2. **기사/안전도우미 정보가 학부모 앱에 안 보인다.** 오늘 우리 아이를 태우는 기사님이 누구인지, 안전도우미는 누구인지, 연락처는 뭔지 전혀 안 나온다. `DailySchedule`에 `vehicle_id`만 있고 기사 정보가 없다. 낯선 사람이 아이를 태우는데 확인할 방법이 없다.

3. **긴급 연락 버튼이 없다.** 아이가 안 내렸다거나 연락이 안 될 때, 앱에서 바로 긴급 신고할 수 있는 기능이 없다. 112 전화 말고 학원이나 운영사에 빠르게 연락할 수 있는 핫라인이 필요하다.

4. **WebSocket 연결 끊김 시 "연결 중..." 또는 "폴링 모드"라고만 뜨는데, 이게 뭔지 모르겠다.** 기술 용어라서 무섭다. "잠시 위치 업데이트가 지연될 수 있습니다" 같은 안내가 나으면 좋겠다. (`MapScreen.tsx:169` — "폴링 모드"는 학부모가 이해 못 함)

5. **로그인이 전화번호+이름만으로 된다.** OTP 인증 없이 devLogin으로 바로 들어가는 건 개발 모드라지만, 실서비스에서 아이 위치를 아무나 볼 수 있으면 큰 문제다. (`LoginScreen.tsx:46` — `devLogin` 사용)

### 요청 사항

1. 홈 화면 카드를 탭하면 상세 화면으로 이동 — 학원명, 차량번호, 기사 이름/사진, 예상 도착 시간, 현재 위치 표시
2. 아이가 하차 후 학원 도착 확인 알림 (학원 출석 연동 또는 안전도우미 확인)
3. 지도에 우리 아이 픽업 지점 마커 + 버스까지 남은 시간(분) 표시
4. 긴급 연락 버튼 (학원/운영사/112 원터치)
5. 자녀 프로필 관리 화면 (알레르기, 특이사항, 비상연락처)

## 학부모B (박성호, 42세) 피드백

> 초등 5학년 아들, 3학년 딸. 수학+피아노+미술 3곳 셔틀. 바쁜 직장인이라 빠른 확인이 중요. 요금 민감.

### 좋은 점

1. **탭 구조가 심플하다.** 홈/스케줄/지도/청구서/프로필 5개 탭이면 충분하다. 쓸데없이 복잡하지 않아서 좋다. (`ParentTabNavigator.tsx:27-53`)

2. **청구서 화면의 미납 요약이 한눈에 보인다.** 상단에 "미납 2건 / 150,000원" 이런 식으로 빨간색 배너로 떠서, 열어보자마자 내가 얼마나 밀렸는지 바로 안다. (`BillingScreen.tsx:47-71`)

3. **청구서 아코디언 UI가 괜찮다.** 접었다 펴서 상세 보는 게 직관적이다. 탑승 횟수, 청구 금액, 납부 기한이 깔끔하게 나온다. (`BillingScreen.tsx:90-183`)

4. **날짜 네비게이션으로 이전/다음 날 스케줄을 쉽게 넘겨볼 수 있다.** 어제 스케줄도 확인 가능하고 내일 것도 볼 수 있어서 편하다. (`ScheduleScreen.tsx:139-181`)

5. **풀 다운 새로고침이 된다.** 회사에서 잠깐 열어보고 아래로 당기면 바로 최신 데이터가 나온다.

### 불편한 점

1. **자녀별 필터가 없다.** 아이가 둘이고 학원이 3곳인데, 홈 화면과 스케줄 화면에서 모든 스케줄이 섞여서 나온다. 아들 것만 보고 싶을 때, 딸 것만 보고 싶을 때 필터링이 안 된다. 6건이 한꺼번에 나오면 스크롤 해야 하는데 바쁜데 짜증난다.

2. **청구서에서 어떤 자녀/어떤 학원 비용인지 구분이 안 된다.** Invoice 데이터에 `student_id`와 `academy_id`가 있지만 화면에는 `billing_month`만 표시된다. (`BillingScreen.tsx:104`) 아이 둘, 학원 3곳이면 월 청구서가 여러 건인데 어느 학원 어느 아이 비용인지 알 수가 없다.

3. **결제 수단이 불분명하다.** "결제하기" 버튼을 누르면 Alert 확인창만 뜨고 바로 결제된다. 카드 등록은 어디서? 자동이체 설정은? 매달 일일이 수동 결제해야 하나? (`BillingScreen.tsx:232-276` — `confirmPayment("test_key", ...)` 시뮬레이션)

4. **요금 상세 내역이 없다.** 탑승 횟수 × 단가 = 총액 같은 상세 내역이 없다. 그냥 "총 15회, 150,000원"만 나온다. 편도인지 왕복인지, 단가가 얼마인지, 할인이 적용됐는지 모르겠다.

5. **이전 달 청구서 조회가 불편하다.** 월별 필터나 기간 선택이 없고 전체 청구서가 한 리스트로 나온다. 6개월 치가 쌓이면 스크롤이 길어진다.

6. **알림 설정을 바꿀 수 없다.** 탑승/하차 알림만 받고 싶은데, 알림 종류별로 온/오프 할 수 있는 설정이 없다. 미납 독촉 알림은 끄고 싶을 수도 있는데.

7. **홈 화면에 "완료" 상태 스케줄이 계속 남아있다.** 이미 다 끝난 건 접어두거나 아래로 밀어주면 좋겠다. 아침에 끝난 태권도 스케줄이 저녁까지 떠 있으면 혼란스럽다.

### 불안한 점

1. **운행 지연/비정상 상황 알림이 없다.** 셔틀이 평소보다 15분 늦게 오는데 아무 알림이 없다. 지도에서 직접 확인해야 하는데, 회의 중에 앱을 계속 보고 있을 수 없다. 자동으로 "예정 시간 대비 10분 이상 지연" 알림이 왔으면 좋겠다.

2. **결제 실패 시 아이 탑승이 거부될 수 있나?** 미납이 쌓이면 어떻게 되는지 안내가 없다. 아이가 갑자기 못 타게 되면 곤란하다.

3. **아이 두 명이 다른 차를 타는데 지도에서 구분이 안 된다.** 버스 마커가 여러 개 뜰 텐데, 어떤 버스가 아들 버스이고 어떤 게 딸 버스인지 구분할 수 없다. 마커에 자녀 이름이나 학원 이름이 표시되면 좋겠다.

4. **앱을 안 쓰는 배우자도 알림을 받을 수 있나?** 내가 회의 중이면 아내가 확인해야 하는데, 보호자 계정을 2개 연결할 수 있는지 모르겠다. API에 `guardian_id`는 하나만 있다. (`students.ts:6`)

### 요청 사항

1. 자녀별 필터 탭 (전체/아들/딸) — 홈, 스케줄, 청구서 전 화면에 적용
2. 청구서에 자녀 이름 + 학원 이름 표시, 탑승 상세 내역 (날짜별 탑승 기록)
3. 자동결제(자동이체/카드 등록) 기능
4. 운행 지연 자동 알림 (예정 시간 대비 N분 이상 늦을 때)
5. 보조 보호자 등록 기능 (배우자/조부모도 알림 수신)
6. 지도에서 버스 마커에 자녀 이름 또는 학원 이름 표시
