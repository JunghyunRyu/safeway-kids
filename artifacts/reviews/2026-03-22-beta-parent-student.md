## 새 기능 UX 평가

### 1. SOS/긴급 버튼 (ITEM-A01)

**학부모A 김지영**: "앱 오른쪽 아래에 빨간 SOS 버튼이 생겼다. 한번 눌러보니 '긴급 상황입니까?' 확인 팝업이 뜨고, 확인하면 112로 전화가 바로 연결된다고 한다. 위치 정보도 같이 보내준다니 안심이 된다. 다만 아이 화면에서 실수로 누를까 걱정은 된다 — 확인 팝업이 있으니 괜찮을 것 같긴 하지만."

**학부모B 박성호**: "SOS 버튼 위치가 좋다. 어떤 화면에서든 바로 접근할 수 있다. 기사 앱에서는 사고 유형(차량사고/학생부상/차량고장/기타)을 선택할 수 있다는 것도 적절하다."

**학생C 정민서**: "SOS 버튼이 생겼다! 빨간색이라 눈에 잘 띄고, 진짜 위급할 때 누르면 될 것 같다. 근데 SOS라는 글씨가 좀 작다 (10px). 조금만 더 크면 좋겠다."

**UX 평가**: 양호. 2단계 확인으로 오탑 방지, GPS 위치 첨부, 관리자 알림 + 112 연결 모두 적절. SOS 라벨이 `fontSize: 10`으로 다소 작지만 아이콘(`warning`)과 빨간 배경으로 충분히 인식 가능.

### 2. 미탑승(No-Show) 처리 (ITEM-A04)

**학부모A 김지영**: "기사님이 미탑승 처리를 하면 알림이 온다고? 그건 좋다. 갑자기 아이가 안 탔는데 연락이 없으면 불안하니까. 근데 실제로 어떤 알림이 오는지 확인해봐야 할 것 같다."

**UX 평가**: 기사 앱 `RouteScreen.tsx:203-217`에 "미탑승" 버튼이 `scheduled` 상태에서 "탑승" 버튼 옆에 표시됨. 확인 팝업 있음. 백엔드에서 `_send_no_show_notification`으로 학부모+학원 관리자 알림 발송. API `markNoShow` 함수도 구현됨 (`schedules.ts:92-94`). 적절하게 구현됨.

### 3. 탑승/하차 확인 팝업 (ITEM-A05)

**학부모A 김지영**: "기사님이 탑승/하차 버튼을 누를 때 확인 팝업이 뜬다고? 실수 방지 차원에서 좋다."

**UX 평가**: `RouteScreen.tsx:101-106`에서 탑승 시 `Alert.alert("탑승 확인", ...)`, 하차 시 `Alert.alert("하차 확인", ...)` 확인 팝업 구현. 또한 5분 이내 "탑승 되돌리기"/"하차 되돌리기" 기능도 추가됨 (`undoBoard`, `undoAlight`). UX 적절.

### 4. 차량 내 잔류 확인 (ITEM-A03)

**학부모A 김지영**: "차량 점검 완료 기능이 생겼다니 세림이법 걱정이 좀 줄었다. 실제로 기사님이 꼼꼼히 확인해주실지는 모르겠지만, 시스템적으로 강제하는 게 중요하다."

**UX 평가**: `RouteScreen.tsx:465-472`에서 모든 활성 스케줄 완료 시 "차량 점검 완료" 버튼 표시. `submitVehicleClearance` API 구현 (`schedules.ts:107-118`). 다만 현재 체크리스트가 Alert 한 번 확인으로 `seats_checked: true, trunk_checked: true, locked: true` 일괄 전송되는 형태라, 기획서의 "3개 항목 개별 체크" 요건과 다소 차이. 체크리스트 모달이 아닌 단순 Alert 확인으로 구현됨.

### 5. 기사/차량 정보 API 확장 (ITEM-A02, B13)

**학부모A 김지영**: "여전히 홈 화면 카드에 기사 이름이나 차량번호가 안 보인다. API에서 내려온다고 해도 화면에 안 나오면 의미가 없다."

**학부모B 박성호**: "스케줄 카드에 학원 이름도 안 나온다. 데이터는 있다면서요? 왜 안 보여주는 겁니까?"

**UX 평가**: 백엔드와 API 타입은 완비되었으나 **프론트엔드 렌더링 누락**. `HomeScreen.tsx`의 `ScheduleCard`는 여전히 `studentName`, `status`, `pickupTime` 3개 필드만 표시. `ScheduleScreen.tsx`의 `ScheduleItem`도 동일하게 `driver_name`, `academy_name`, `vehicle_license_plate` 미표시. 학부모가 가장 원했던 정보가 화면에 나오지 않는 상태.

### 6. 학생 사진 표시 (ITEM-B06)

**UX 평가**: 기사 앱 `RouteScreen.tsx:150-159`에서 `student_photo_url`이 있으면 프로필 사진 표시, 없으면 순번 인덱스 원 표시. 기사 앱에서는 적절히 구현됨. 학부모 앱에서는 학생 사진 미표시.

### 7. devLogin UI 제거 (ITEM-B10)

**학생C 정민서**: "로그인 화면이 깔끔해졌다. '개발 모드 로그인'이 안 보이고, 전화번호 넣고 인증번호 받는 방식으로 바뀌었다. 정상적인 앱처럼 보인다."

**UX 평가**: `__DEV__` 분기 적절. 프로덕션: OTP 2단계 (전화번호 → 인증번호+이름+역할), 개발: 기존 devLogin 유지. 다만 `ROLE_OPTIONS`에 `student` 역할이 빠져있어서, 학생이 프로덕션 빌드에서 로그인할 수 없는 문제 발견.

### 해결된 항목 (4건)
1. SOS/긴급 버튼 — 전 앱 적용, 모든 역할 접근 가능

### 핵심 미반영 사항
**가장 시급한 것은 이미 API에 존재하는 데이터를 화면에 보여주는 것**:
- `HomeScreen.tsx` ScheduleCard에 `academy_name`, `driver_name`, `vehicle_license_plate` 표시
- `ScheduleScreen.tsx` ScheduleItem에 동일 정보 표시
- `student/ScheduleScreen.tsx`에서 `"오늘의 등원"` → `academy_name` 치환

이 3개 화면 변경만으로 학부모/학생 만족도가 유의미하게 상승할 것으로 예상.
