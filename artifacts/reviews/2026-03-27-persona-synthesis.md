## 2. 공통 감점 패턴 분석

### TIER 1 — 치명적 감점 (전원 해당, 총 +15~20점 효과)

#### GAP-A: 개인정보 보호 체계 완전 부재
**지적한 페르소나:** 학부모(11/20), 정책심사위원(11/20), 원장, 지자체, 기술심사위원
**문제:** 미성년자 얼굴이라는 민감 생체 정보를 다루면서:
- 개인정보 처리방침 없음
- 보호자 동의 절차 없음
- 데이터 보관/파기 정책 없음
- PIA(Privacy Impact Assessment) 계획 없음
- 유출 대응 계획 없음

**해소 전략:**
1. Tech Spec에 "개인정보 보호 설계" 섹션 추가
2. 데모에 "보호자 동의 절차" 화면 포함 (동의서 UI)
3. 데이터 보관: 얼굴 임베딩만 저장 (원본 이미지 미보관), AES-256 암호화
4. 보관 기간: 재학 기간 + 30일 후 자동 파기
5. PIA 요약서를 데모 부록으로 첨부
6. **예상 점수 개선:** 학부모 +7, 정책심사위원 +7, 원장 +3, 지자체 +3

#### GAP-B: 하차 인식 시나리오 부재
**지적한 페르소나:** 학부모, 인솔교사
**문제:** 승차만 있고 하차 확인이 없음 → "절반의 안전"
**해소 전략:**
1. 시나리오 ①을 "승하차 안면인식"으로 확장 (승차 모드 + 하차 모드)
2. 하차 시 등록 명단 대비 하차 체크 → 미하차 인원 알림
3. 하차 완료 시 학부모에게 "OOO 원생 하차 확인" 푸시
4. **예상 점수 개선:** 학부모 +3, 인솔교사 +2

### TIER 2 — 주요 감점 (3명 이상 해당, 총 +8~12점 효과)

#### GAP-C: 비용/ROI 정보 부재
**지적한 페르소나:** 원장(12/20), 지자체, 정책심사위원
**해소 전략:**
1. Tech Spec에 "비용 분석" 섹션 추가
2. 차량 1대당: 설치비 200~300만원, 월 이용료 10~15만원 (원장 수용 범위 참고)
3. ROI: 인솔교사 인건비(월 120~150만원) 완전 대체는 아니나 보조 역할로 시간 절감
4. 사회적 비용: 통학차량 사고 1건당 사회적 비용 약 3~5억원 (보험연구원 자료)
5. **예상 점수 개선:** 원장 +4, 지자체 +2, 정책심사위원 +2

#### GAP-D: 수치적 근거 및 성능 검증 프로토콜
**지적한 페르소나:** 기술심사위원(12/20), 정책심사위원
**해소 전략:**
1. 성능 검증 프로토콜 수립 (NIST FRVT 참조)
2. FAR(False Acceptance Rate) / FRR(False Rejection Rate) 분리 보고
3. End-to-End 파이프라인 타이밍 다이어그램
4. 벤치마크 테스트 결과를 데모에 포함 (실측 수치)
5. **예상 점수 개선:** 기술심사위원 +8, 정책심사위원 +2

#### GAP-E: 확장성 로드맵 (PoC → 상용화)
**지적한 페르소나:** 기술심사위원(11/20), 지자체, 정책심사위원
**해소 전략:**
1. 3단계 로드맵: Phase 1 PoC → Phase 2 파일럿(Jetson 5대) → Phase 3 전국 확산
2. HAL(Hardware Abstraction Layer) 설계로 CPU↔Jetson 전환 가능성 입증
3. Jetson Orin Nano 기준 추론 시간 예상치 (30~50ms, 20+ FPS)
4. Fleet 관리 아키텍처 간략도
5. **예상 점수 개선:** 기술심사위원 +5, 지자체 +3, 정책심사위원 +2

### TIER 3 — 현장 적합성 (운전기사, 인솔교사, 총 +5~8점 효과)

#### GAP-F: 운행/정차 모드 자동 구분
**지적한 페르소나:** 운전기사
**해소 전략:**
1. GPS/OBD 연동 또는 시뮬레이션으로 운행 중/정차 중 자동 전환
2. 정차 중 서 있는 것은 정상 → 이상행동 감지 비활성화
3. **예상 점수 개선:** 운전기사 +3

#### GAP-G: 거짓 경고 억제 메커니즘
**지적한 페르소나:** 운전기사(14/20), 인솔교사
**해소 전략:**
1. 이상행동 감지 지속시간 필터: 2초 이상 지속 시에만 경고
2. 5프레임 → 10프레임 temporal smoothing
3. Confidence threshold 동적 조정
4. **예상 점수 개선:** 운전기사 +3, 인솔교사 +1

#### GAP-H: 현장 전용 UI (인솔교사/운전기사)
**지적한 페르소나:** 인솔교사(14/20), 운전기사, 원장
**해소 전략:**
1. 인솔교사 전용 뷰: 큰 버튼 3개(승차/운행/하차) + 경고 팝업
2. 운전기사 전용 뷰: 최소 정보 + 음성/LED 경고 중심
3. 데모에서는 "현장 UI 프리뷰" 탭으로 시뮬레이션
4. **예상 점수 개선:** 인솔교사 +3, 운전기사 +2, 원장 +2

#### GAP-I: Human Fallback (시스템 장애 시 수동 절차)
**지적한 페르소나:** 학부모, 인솔교사, 운전기사
**해소 전략:**
1. AI 장애 감지 → "수동 확인 모드" 자동 전환
2. 인솔교사에게 수동 출석 체크리스트 표시
3. 학부모에게 "현재 수동 확인 모드입니다" 알림
4. **예상 점수 개선:** 학부모 +3, 인솔교사 +1

## 4. Final Tech Spec에 반드시 포함할 신규 섹션

1. **개인정보 보호 설계** — PIA, 동의 절차, 암호화, 보관/파기
2. **하차 인식 시나리오** — 시나리오 ① 확장
3. **비용 분석 및 ROI** — 설치비, 월정액, 사회적 비용 절감
4. **성능 검증 프로토콜** — NIST FRVT 참조, FAR/FRR, E2E 타이밍
5. **상용화 로드맵** — PoC → 파일럿 → 전국 확산 3단계
6. **HAL(Hardware Abstraction Layer)** — CPU↔Jetson 전환 설계
7. **운행 모드 자동 전환** — GPS/OBD 연동 시뮬레이션
8. **거짓 경고 억제** — 지속시간 필터, temporal smoothing
9. **현장 전용 UI** — 인솔교사/운전기사 뷰
10. **Human Fallback** — AI 장애 시 수동 전환 절차
11. **행정 시스템 연동** — 운행일지, 데이터 내보내기
12. **알림 다중 채널** — 푸시 → SMS → 카카오톡
13. **규제 특례 요청 목록** — 필요한 규제 완화/특례 사항 명시
14. **데모 영상 녹화 계획** — 시나리오별 촬영 가이드
