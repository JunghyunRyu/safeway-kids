## 1. 안전성 입증 (20점 만점)

**점수: 15 / 20**

### 잘된 점
- 4개 시나리오(승하차 안면인식, 이상행동 감지, 잔류인원 감지, 사각지대 감지)가 도로교통법 제53조의 핵심 안전 의무사항(승하차 확인, 운행 중 안전, 하차 후 점검, 주변 안전)에 1:1로 대응되어 있어 **법규 기반 설계 의도**가 명확하다.
- 잔류인원 감지(시나리오 3)는 실제 어린이 통학차량 사고 중 가장 치명적인 "갇힘 사고"를 직접 겨냥하고 있어, 사회적 필요에 부합한다.
- 사각지대 감지(시나리오 4)에서 LiDAR + 초음파 센서 융합 접근은 단일 센서 한계를 보완하는 적절한 설계이다.

### 감점 사유 (-5점)
1. **사고 예방 효과의 정량적 근거 부재 (-3점):** "이 기술이 사고를 예방한다"는 주장은 있으나, 기존 통학차량 사고 통계(연간 사고 건수, 유형별 비율)와 본 시스템 적용 시 예상 감소율에 대한 논리적 추론이 전혀 없다. 심사위원은 "감지한다"가 아니라 "예방한다"는 증거를 요구한다.
2. **안전 체인(Safety Chain) 불완전 (-2점):** 감지 → 경고까지는 설계되어 있으나, 경고 → 실제 조치(차량 정지, 문 잠금 해제 등)로 이어지는 물리적 안전 체인이 명시되지 않았다. PoC 범위 밖이더라도 "상용화 시 이렇게 연결된다"는 로드맵이 있어야 한다.

## 2. 법적 적합성 (20점 만점)

**점수: 11 / 20**

### 잘된 점
- 도메인 4개가 도로교통법 제53조 및 어린이안전관리법의 의무사항과 잘 매핑된다.
- 마스크 착용 대응(70% 인식률 목표)은 감염병 대응 상황을 고려한 것으로 현실적이다.

### 감점 사유 (-9점)
1. **개인정보 처리 계획 완전 누락 (-4점):** 안면인식은 생체정보(개인정보보호법 제23조 민감정보)에 해당한다. 어린이의 안면 데이터 수집, 저장, 처리, 파기에 대한 법적 근거와 동의 절차가 전혀 기술되어 있지 않다. 이것은 심사에서 **즉시 지적**되는 사항이다.
   - 법정대리인(보호자) 동의 절차는?
   - 안면 임베딩 데이터의 저장 위치와 암호화 방안은?
   - 데이터 보존 기간과 파기 절차는?
   - 제3자 제공(학원 → 플랫폼) 시 동의 범위는?

2. **규제 특례 필요 사항 미명시 (-3점):** 규제 샌드박스 신청의 핵심은 "현행법에서 허용되지 않는 부분이 무엇이고, 어떤 특례가 필요한가"를 명확히 하는 것이다. 이 계획서에는:
   - 어떤 규제의 특례를 요청하는지 목록이 없다
   - 특례 기간 및 범위가 정의되지 않았다
   - 특례 종료 후 평가 기준이 없다

3. **아동 데이터 특별 보호 조치 미언급 (-2점):** 14세 미만 아동의 개인정보는 개인정보보호법 제22조에 따라 법정대리인 동의가 필수이며, EU GDPR 수준의 보호가 요구되는 추세이다. 이에 대한 인식이 계획서에 보이지 않는다.

## 3. 사회적 가치 (20점 만점)

**점수: 16 / 20**

### 잘된 점
- 어린이 통학차량 안전은 **국민적 공감대**가 매우 높은 주제로, 기술 적용의 사회적 명분이 강하다.
- 4개 시나리오 모두 실제 사고 사례(갇힘 사고, 하차 시 차량 충돌, 운행 중 부상, 사각지대 사고)에 직접 대응한다.
- 학부모 실시간 알림 연동은 보호자 불안감 해소에 직결되는 기능이다.
- 비접촉 인증(카메라 앞에 서면 자동 인식)은 어린이 사용 편의성과 위생을 동시에 고려한 설계이다.

### 감점 사유 (-4점)
1. **기존 대안 대비 우위 분석 없음 (-2점):** 현재 시행 중인 "동승보호자 의무 탑승" 제도, RFID 태그 기반 승하차 확인 시스템 등 기존 대안과 비교하여 AI 기반 시스템의 우위를 설명하지 않았다. "왜 AI여야 하는가?"에 대한 답이 필요하다.
2. **사회적 우려 대응 부재 (-2점):** 어린이 안면인식에 대한 사회적 우려(감시 사회 논란, 데이터 유출 시 피해)에 대한 선제적 대응이 없다. 심사위원 중 시민단체 출신이 있다면 반드시 질문할 사항이다.

## 4. 실현 가능성 (20점 만점)

**점수: 14 / 20**

### 잘된 점
- i5 CPU에서 동작하는 PoC라는 현실적 접근이 좋다. "고가 장비 없이도 기술 검증 가능"이라는 메시지를 줄 수 있다.
- ONNX Runtime 기반 추론, ArcFace 임베딩 등 기술 선택이 합리적이다.
- Standalone 모드 + 백엔드 자동 감지는 데모 안정성을 높인다.
- 원클릭 실행(start.bat)은 비기술자 시연에 매우 중요한 요소이다.

### 감점 사유 (-6점)
1. **PoC → 파일럿 → 상용화 로드맵 부재 (-3점):** 심사위원이 가장 궁금한 것은 "이 PoC가 성공하면 다음에 뭘 할 것인가"이다. 구체적으로:
   - 파일럿 대상 학원/차량 수, 기간, 지역
   - 실제 하드웨어(Jetson, LiDAR) 도입 일정과 비용
   - 사업 모델(B2B? B2G? 구독형?)
   - 파일럿 성공 기준(KPI)

2. **실 환경과 PoC 간 Gap 인정 부족 (-2점):** 계획서에 Non-goals로 하드웨어 통합을 제외했으나, 심사위원 관점에서는 "웹캠으로 한 것을 실제 차량 환경에서도 할 수 있는가?"에 대한 기술적 브릿지 설명이 필요하다. 예를 들어:
   - 차량 진동, 온도 변화, 역광 등 실환경 변수 대응 계획
   - 실시간성(5 FPS → Jetson에서 30 FPS) 향상 경로

3. **비용 추정 없음 (-1점):** 차량 1대당 하드웨어 + 소프트웨어 비용 추정이 없으면 "이게 현실적으로 보급 가능한가?"라는 질문에 답할 수 없다.

## 5. 데모 설득력 (20점 만점)

**점수: 16 / 20**

### 잘된 점
- 스텝 위저드 패턴으로 4개 시나리오를 순차적으로 보여주는 구조는 비기술 심사위원에게 이해하기 쉽다.
- 프레젠터 가이드 UI(각 시나리오 설명 패널)는 시연자가 설명할 포인트를 놓치지 않게 해준다.
- 실시간 성능 대시보드(CPU, 메모리, FPS, 추론 시간)는 기술의 실제 동작을 증명하는 데 효과적이다.
- 웹캠 실패 시 사전 녹화 영상 fallback은 데모 사고 방지에 필수적인 좋은 설계이다.
- 한국어 UI + 영문 기술 용어 병기 결정은 심사위원 친화적이다.

### 감점 사유 (-4점)
1. **"시뮬레이션"이라는 인상 극복 전략 부재 (-2점):** 사각지대 시나리오는 전적으로 소프트웨어 시뮬레이션이며, 안면인식/이상행동/잔류인원도 웹캠 기반이다. 심사위원이 "이건 그냥 데모 아닌가? 실제 차량에서도 되나?"라는 의문을 가질 때 설득할 브릿지 메시지가 설계에 포함되어 있지 않다.
   - 예: 각 시나리오 화면에 "실제 배포 시 구성" 다이어그램을 작은 패널로 함께 보여주기

2. **End-to-End 스토리라인 약함 (-2점):** 4개 시나리오가 독립적으로 설계되어 있어 "등원 → 운행 → 하원"이라는 하루의 통학 흐름으로 묶이는 내러티브가 부족하다. 심사위원에게는 기술 기능 나열보다 **스토리**가 설득력 있다.
   - 예: "7:30 등원 시작 → 안면인식 탑승 확인 → 8:00 운행 중 이상행동 감지 → 8:20 학원 도착 하차 → 잔류인원 점검 → (별도) 하교 시 사각지대 감지"

## 90점 이상 달성을 위해 반드시 보강해야 할 사항

### [필수 1] 개인정보 처리 체계 수립 (예상 +8점)

**현재 문제:** 어린이 안면 데이터(생체 민감정보)에 대한 법적 처리 체계가 전무하다.

**보강 내용:**
1. **개인정보 영향평가(PIA) 요약서** 작성 — 수집 항목, 처리 목적, 보존 기간, 파기 방법 명시
2. **법정대리인 동의서 양식** 제시 — "안면 데이터 수집 및 활용에 대한 보호자 동의"
3. **데이터 보호 기술 조치** 명시:
   - 안면 임베딩 벡터는 원본 얼굴 복원이 불가능한 단방향 변환 (이 점을 강조)
   - 로컬 디바이스 처리 원칙 (클라우드 전송 최소화)
   - AES-256 암호화 저장
   - 학기 종료 시 자동 파기
4. **데모 시 설명 슬라이드:** "안면인식 ≠ 감시, 안전 목적 한정 사용" 메시지

### [필수 2] 규제 특례 요청 사항 명시 (예상 +5점)

**보강 내용:**
1. 특례 요청 목록 작성:
   - 어린이 통학차량 내 영상 장치 설치 기준 특례 (현행법은 블랙박스 중심)
   - 생체정보 처리 목적 확대 특례 (안전 목적 안면인식)
   - 자동화된 승하차 확인 기록의 법적 효력 인정
2. 특례 기간: 2년 (파일럿 기간 포함)
3. 특례 종료 후 평가 기준: 사고 감소율, 오작동률, 보호자 만족도
4. 안전 장치: 동승보호자 병행 운영 (AI 단독 대체 아님)

### [필수 3] 사고 예방 효과 정량 근거 (예상 +3점)

**보강 내용:**
1. 최근 3년간 어린이 통학차량 사고 통계 인용 (경찰청/도로교통공단 자료)
2. 사고 유형별(갇힘, 하차 시 충돌, 운행 중 부상, 사각지대) 비율 분석
3. 본 시스템 적용 시 예상 감소율 논리적 추론:
   - "잔류인원 감지 정확도 99% → 갇힘 사고 95% 이상 예방 가능" (기존 육안 확인 누락률 대비)
   - "사각지대 감지 → 하차 후 충돌 사고 80% 이상 예방 가능"
4. 해외 유사 사례 인용 (미국 School Bus Stop-Arm Camera 법제화 사례 등)

### [필수 4] 상용화 로드맵 (예상 +3점)

**보강 내용:**
1. **Phase 1 (PoC, 현재):** 소프트웨어 시뮬레이션 → 기술 검증
2. **Phase 2 (파일럿, 6개월):** 수도권 학원 3곳, 차량 10대 실차 장착
   - Jetson Orin Nano + 광각 카메라 + LiDAR 장착
   - 비용 추정: 차량 1대당 하드웨어 150만원 + 월 구독 5만원
3. **Phase 3 (확대, 12개월):** 50대 차량, 전국 3개 권역
4. **Phase 4 (상용화, 24개월):** SaaS 모델로 전환, 지자체 보조금 연계
5. 각 단계별 성공 기준(KPI) 명시

### [권장 5] 데모 스토리라인 강화 (예상 +2점)

**보강 내용:**
- 4개 시나리오를 "등원 → 운행 → 도착 → 하교" 하루 일과 흐름으로 재구성
- 각 시나리오 화면에 "실제 배포 시 하드웨어 구성" 소형 다이어그램 포함
- 도입부에 30초짜리 "현재 문제점 → 우리의 해결책" 인트로 화면 추가

## 심사위원으로서 반드시 물을 질문 5개 + 기대 답변

### Q1. "어린이 안면 데이터는 민감 정보인데, 개인정보보호법과의 정합성은 어떻게 확보하셨나요?"

**기대 답변:**
> "안면 데이터는 원본 이미지가 아닌 512차원 수치 벡터(임베딩)로 변환하여 저장합니다. 이 벡터에서 원본 얼굴을 복원하는 것은 기술적으로 불가능합니다. 또한 모든 처리는 차량 내 엣지 디바이스에서 이루어지며, 클라우드로 안면 데이터를 전송하지 않습니다. 법정대리인의 별도 동의를 받으며, 학기 종료 시 자동 파기됩니다. 개인정보 영향평가(PIA)도 완료할 예정입니다."

### 강점

### 근본적 문제
그러나 이 계획서는 **개발자가 개발자를 위해 쓴 기술 문서**이다. 규제 샌드박스 심사위원이 보는 것은 기술 사양이 아니라 **안전 정책 적합성, 법적 정합성, 사회적 가치, 상용화 실현성**이다. 현재 계획서에는:

1. **개인정보보호법 대응이 전무하다** — 이것만으로도 심사 탈락 사유가 될 수 있다
2. **규제 특례 요청 사항이 불명확하다** — 심사위원은 "뭘 허용해 달라는 건지" 모른다
3. **상용화 로드맵이 없다** — "PoC 이후에 어떻게?"가 공백이다
4. **사고 예방 효과의 정량 근거가 없다** — "좋은 기술"과 "안전을 입증한 기술"은 다르다
