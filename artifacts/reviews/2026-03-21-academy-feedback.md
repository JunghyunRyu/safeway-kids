# 학원 관리자 페르소나 리뷰

> 작성일: 2026-03-21
> 리뷰 대상: 웹 대시보드 (학원 관리자 + 플랫폼 관리자), 모바일 앱 관리자 화면
> 리뷰 방법: 코드 기반 분석 (web/src/pages, web/src/components, mobile/src/screens/admin, backend models/routers)

---

## 학원A: 정은서 원장 (태권도 학원, 학생 80명, 차량 2대)

> "디지털 전환에 관심은 있지만 복잡한 건 싫어요. 지금은 기사님한테 전화해서 배차하고, 학부모한테 카톡으로 안내하는데... 민원이 너무 많아요."

### 전반적 인상

처음 로그인 화면을 보니 깔끔하긴 한데, **학원 등록 과정이 보이지 않아요**. 로그인하면 "등록된 학원이 없습니다"라고 뜨는데, 그 다음에 뭘 해야 하는지 안내가 없습니다. 학원 등록은 플랫폼 관리자가 해줘야 하는 건지, 제가 직접 할 수 있는 건지 모르겠어요.

### 시나리오별 평가

#### 1. 학원 가입/초기 설정 [점수: 2/5]
- **문제**: 학원 관리자 로그인 후 학원이 없으면 "등록된 학원이 없습니다" 노란 경고만 뜸. 셀프 등록 UI가 없음
- **문제**: 학원 등록은 PlatformAcademiesPage에서 플랫폼 관리자만 할 수 있음 → 온보딩 병목
- **요청**: 학원 관리자가 로그인하면 바로 학원 정보를 입력하고, 플랫폼 측 승인을 받는 플로우가 있으면 좋겠음
- **요청**: 초기 설정 마법사 (학원정보 → 차량등록 → 학생등록 → 스케줄설정) 같은 단계별 안내 필요

#### 2. 학생 등록 [점수: 3.5/5]
- **좋은 점**: 개별 학생 등록 폼이 간단함 (이름, 생년월일, 학년, 보호자 연락처). 검색/정렬/엑셀 내보내기 있음
- **문제**: 엑셀 대량 업로드(PlatformUploadPage)가 **플랫폼 관리자 전용 메뉴**에만 있음. 학원 관리자 사이드바에 없음!
- **문제**: 80명을 하나하나 등록할 수는 없음. 학원 관리자도 엑셀 업로드를 직접 할 수 있어야 함
- **문제**: 보호자 연락처를 입력하지만 payload에서 실제로 전송되지 않음 (코드상 form.guardian_phone이 POST body에 없음)
- **문제**: 학생 상세에서 "보호자 ID"만 UUID로 보이고, 보호자 이름/연락처가 안 보임
- **요청**: 학생 등록 시 보호자 자동 연결 (전화번호로 기존 학부모 검색 or 신규 보호자 초대)

#### 3. 셔틀 스케줄 설정/변경 [점수: 2/5]
- **문제**: 스케줄 페이지(SchedulesPage)에서 볼 수 있는 건 날짜별 조회와 "일일 파이프라인 실행" 뿐
- **문제**: **스케줄 템플릿(ScheduleTemplate) 관리 UI가 없음**. 백엔드에 schedule_templates 테이블이 있고 요일별 반복 스케줄을 저장하는데, 이걸 설정하는 웹 화면이 없음
- **문제**: "파이프라인"이라는 용어가 학원 운영자에게 낯설음. "오늘 배차 자동 생성" 같은 표현이 더 나음
- **문제**: 스케줄 테이블에 학생 이름이 안 보이고 student_id의 앞 8글자만 표시됨. 누구 스케줄인지 알 수 없음
- **요청**: "월~금 오후 3시 픽업" 같은 반복 스케줄을 학생별로 설정하는 화면 필요
- **요청**: 특정 날짜 스케줄 취소/변경 기능 (예: 공휴일, 학생 결석)
- **요청**: 빠른 작업에 "일일 파이프라인 실행"이라고 되어 있는데, 개발 용어 대신 "오늘 배차 생성"으로 변경 요망

#### 4. 차량/기사 배정 [점수: 3/5]
- **좋은 점**: 차량 등록 (번호판, 정원, 모델명) 깔끔함. CRUD 모두 가능
- **문제**: 차량에 기사를 배정하는 UI가 없음. 기사(driver) 관리가 학원 관리자 메뉴에 없음
- **문제**: 차량 수정 API가 `/telemetry/vehicles/:id`인데 조회는 `/vehicles/vehicles` → API 경로 불일치가 에러를 유발할 수 있음
- **요청**: 차량별 담당 기사 배정 기능, 기사 연락처/면허 정보 관리

#### 5. 실시간 차량 위치 모니터링 [점수: 1/5 (학원 관리자 기준)]
- **문제**: 관제 센터(PlatformMapPage)가 **플랫폼 관리자 전용**. 학원 관리자 사이드바에 없음
- **문제**: 학원 관리자는 자기 학원 차량의 실시간 위치를 볼 수 없음
- **요청**: 학원 관리자용 간이 지도 (자기 학원 차량만 표시) 꼭 필요. 이게 핵심 기능인데 빠져 있음

#### 6. 학부모 민원 대응 [점수: 2/5]
- **문제**: 학부모에게 "아이가 탑승했습니다" 알림을 보내는 기능은 백엔드에 FCM/SMS 모듈이 있지만, 학원 관리자가 수동으로 알림을 보내거나 확인하는 UI는 없음
- **문제**: 학부모 앱에서 보내오는 문의/민원을 접수하는 화면이 없음
- **요청**: "오늘 탑승한 학생" / "미탑승 학생" 리스트를 한눈에 볼 수 있어야 함
- **요청**: 학부모에게 공지사항 보내기 기능

#### 7. 월간 요금/청구 관리 [점수: 4/5]
- **좋은 점**: 요금제 CRUD, 청구서 일괄 생성, 결제 처리, 영수증 출력 모두 있음. 건당 요금, 월 상한 설정 가능
- **좋은 점**: 청구서 엑셀 내보내기 있음
- **아쉬운 점**: 학생 ID가 UUID로만 표시되어 어떤 학생 청구서인지 바로 파악 어려움
- **아쉬운 점**: 청구서 중복 생성 경고 메시지는 있지만 실제 중복 방지 로직이 "있을 수 있다"고만 알려줌

#### 8. 컴플라이언스 문서 관리 [점수: 1/5 (학원 관리자 기준)]
- **문제**: 컴플라이언스 메뉴가 플랫폼 관리자 전용. 학원 관리자 사이드바에 없음
- **문제**: 우리 학원의 보험증서, 차량검사증을 직접 올리고 관리할 수 없음
- **요청**: 만료 임박 알림 같은 건 학원 관리자에게 특히 필요한 기능

#### 9. 보고서/통계 [점수: 2/5]
- **좋은 점**: 대시보드에 오늘 스케줄 파이차트, KPI 카드 (등록학생, 차량, 스케줄, 미결제) 있음
- **아쉬운 점**: 주간/월간 운행 보고서가 없음 (총 운행 건수, 정시율, 평균 운행시간 등)
- **아쉬운 점**: 학부모 만족도나 민원 현황을 볼 수 없음

### 정은서 원장 종합 의견

**기존 방식(전화/카톡) 대비 장점**: 학생 정보 관리, 청구서 자동 생성, 차량 등록이 체계적. 매달 엑셀로 정리하던 청구서가 버튼 하나로 나오는 건 확실히 좋음.

**기존 방식 대비 단점**:
- 스케줄 템플릿 설정 UI가 없어서 매일 파이프라인을 돌려야 할 텐데, 그 전에 학생별 반복 스케줄을 어디서 설정하는지 모르겠음
- 학원 관리자가 볼 수 있는 게 너무 적음. 관제센터, 컴플라이언스, 엑셀업로드가 다 플랫폼 관리자 전용
- 기사 관리, 학부모 알림이 없어서 결국 카톡은 계속 써야 할 것 같음

**도입 장벽**: 초기 세팅이 막막함. 학원 등록부터 학생 80명 입력까지 가이드가 없으면 포기할 수 있음.

**비용 대비 가치**: 월 요금에 따라 다르겠지만, 현재 상태로는 청구 관리 + 학생 DB 정도의 가치. 실시간 모니터링과 학부모 알림이 합쳐져야 전화/카톡을 대체할 가치가 생김.

**총점: 55/100**

## 학원B: 김태호 원장 (영어학원, 학생 120명, 차량 3대)

> "엑셀로 관리하고 있는데 한계가 있어요. 비용 절감도 중요하고, 학부모에게 '우리 학원은 디지털 셔틀 관리 시스템을 쓴다'고 어필하고 싶어요."

### 전반적 인상

대시보드 디자인은 깔끔하고 현대적임. teal 컬러 톤이 깔끔하고, 카드 기반 UI가 보기 좋음. 다크모드 지원도 있어서 프리미엄 느낌은 줄 수 있음. 하지만 "실제로 쓸 수 있는가"와 "보여줄 수 있는가"는 다른 문제.

### 시나리오별 평가

#### 1. 학생 대량 등록 [점수: 2/5]
- 120명을 한 명씩 등록하면 반나절 걸림
- 엑셀 업로드(PlatformUploadPage)는 잘 만들어져 있음: 드래그앤드롭, 파일 검증(5MB, xlsx/xls만), 행별 성공/실패 결과, KPI 카드로 요약
- **그런데 이 기능이 플랫폼 관리자 메뉴에만 있음**. 학원 관리자가 직접 못 씀
- 엑셀 템플릿 다운로드가 없음. 어떤 칼럼 순서로 데이터를 넣어야 하는지 안내 없음
- **요청**: 학원 관리자 메뉴에도 엑셀 업로드 추가 + 템플릿 다운로드 버튼

#### 2. 비용 관리 [점수: 3.5/5]
- 요금제 설정 (건당 요금, 월 상한) 유연함. 기본요금제/프리미엄요금제 같은 복수 요금제 가능
- 청구서 일괄 생성 → 결제 확인 → 영수증 출력 워크플로우 완성됨
- **아쉬운 점**: 학부모별 요금제 배정 기능이 대시보드에 안 보임 (어떤 학생이 어떤 요금제인지)
- **아쉬운 점**: 정산 리포트가 없음 (월별 매출 총액, 미수금 추이 그래프)
- **요청**: 월별 매출 대시보드 + 미수금 연체 알림 자동화

#### 3. 학부모 프리미엄 이미지 [점수: 2.5/5]
- 학부모 앱이 별도로 있음 (HomeScreen, ScheduleScreen, MapScreen, BillingScreen) → 학부모에게 "앱 설치하세요"라고 안내 가능
- **좋은 점**: 학부모 앱에 실시간 지도, 스케줄 확인, 청구서 확인이 있음
- **문제**: 탑승/하차 알림이 자동으로 가는지 확인 어려움. FCM 토큰 등록은 있지만 탑승 시 자동 발송 로직이 명시적이지 않음
- **문제**: 학원 관리자가 학부모에게 공지사항이나 안내 메시지를 보낼 수 없음
- **요청**: "오늘 비 때문에 셔틀 10분 지연됩니다" 같은 즉시 공지 기능
- **요청**: 학부모 앱에 학원 브랜딩 (학원 로고, 색상) 커스터마이징

#### 4. 일일 운영 편의성 [점수: 3/5]
- 대시보드에서 오늘 현황 한눈에 파악 가능 (KPI 카드 4개)
- "빠른 작업" 영역에서 주요 기능 접근 가능
- **문제**: 매일 아침 해야 하는 작업 순서가 불명확. "오늘 배차 생성 → 기사에게 배차표 전달 → 운행 모니터링" 순서인데, 이 워크플로우가 가이드되지 않음
- **문제**: 기사 앱에서 배차를 확인하는 건 driver 역할 모바일 앱이 있을 텐데, 학원 관리자가 "기사에게 오늘 배차가 전달되었는지" 확인할 수 없음
- **요청**: "오늘의 체크리스트" 기능 (배차 생성됨? 기사 확인됨? 모든 학생 탑승 완료?)

#### 5. 데이터 마이그레이션 [점수: 2/5]
- 기존 엑셀 데이터를 옮기려면 학생 엑셀 업로드밖에 없음
- 차량, 스케줄, 기사 정보는 수동 입력해야 함
- **요청**: 초기 데이터 마이그레이션 지원 (차량, 기사, 스케줄 템플릿도 엑셀로 한번에)

### 김태호 원장 종합 의견

**경쟁 서비스 대비 차별점**: UI가 현대적이고 다크모드까지 지원하는 건 좋음. 하지만 경쟁사(스쿨버스, 키즈노트 등)와 비교하면 학부모 소통 기능이 약함. 학부모에게 "우리 학원은 AI 경로 최적화로 아이를 안전하게 모십니다"라고 말하려면 학부모 앱에서 실시간 추적이 확실히 되고, 탑승/하차 알림이 자동으로 가야 함.

**비용 대비 가치**: 청구 관리 자동화만으로도 매달 반나절 엑셀 작업이 줄어 가치 있음. 하지만 "프리미엄 이미지"를 주려면 학부모가 체감할 수 있는 기능(실시간 알림, 도착 예정 시간 안내)이 보강되어야 함.

**초기 세팅 예상 시간**: 학원 등록(플랫폼 관리자 요청) + 학생 120명(엑셀 업로드가 직접 가능하다면 30분, 수동이면 3시간 이상) + 차량 3대(10분) + 스케줄 템플릿(UI 없어서 불가). **최소 반나절~1일**.

**총점: 60/100**

## 학원C: 이수진 실장 (대형 종합학원, 학생 300명, 차량 5대)

> "전산 시스템이 이미 있어서, 새로 도입하려면 기존 ERP와 연동이 되거나 최소한 API로 데이터를 주고받을 수 있어야 합니다. 컴플라이언스도 중요해요."

### 전반적 인상

300명 규모 학원에서 쓰기에는 몇 가지 구조적 한계가 보입니다. 기능 자체는 기본기가 잘 갖춰져 있지만, 대규모 운영에 필요한 기능(페이지네이션, 필터링, 권한 분리, API 연동)이 부족합니다.

### 시나리오별 평가

#### 1. 대량 데이터 처리 [점수: 2.5/5]
- 학생 목록이 클라이언트 사이드 필터링. 300명 전체를 한번에 가져오는 구조 → 초기 로딩 느릴 수 있음
- 학생 검색이 이름 필터만 지원. 학년별, 반별, 활성/비활성 필터 없음
- 청구서도 전체 학원의 전체 기간 청구서를 한번에 가져옴 → 300명 x 12개월 = 3,600건
- **요청**: 서버사이드 페이지네이션 + 필터 (학년, 상태, 날짜 범위)
- **요청**: DataTable의 pageSize 옵션은 있지만 서버에서 잘라서 가져오는 건 아님

#### 2. ERP 연동 가능성 [점수: 2/5]
- REST API는 잘 설계되어 있음 (FastAPI, 표준적인 CRUD 엔드포인트)
- **문제**: API 문서가 대시보드에서 직접 접근 불가 (Swagger는 백엔드 /docs에 있겠지만)
- **문제**: Webhook이나 이벤트 알림 기능 없음 → ERP에서 학생 등록되면 자동으로 SafeWay에도 등록되는 연동 불가
- **문제**: 배치 API 부족 → 300명 학생을 한번에 업데이트하는 bulk update API 없음
- **요청**: Webhook 연동 (학생 등록/변경 시 외부 시스템에 콜백)
- **요청**: API 키 기반 인증 (현재 OTP 기반이라 서버간 통신에 부적합)

#### 3. 컴플라이언스 관리 [점수: 3.5/5]
- PlatformCompliancePage에 문서 유형별 관리 잘 되어 있음: 사업자등록증, 보험증서, 차량검사증, 운전면허증, 범죄경력조회
- 만료일 관리 + "만료 임박" 탭으로 갱신 필요 문서 한눈에 파악
- 파일 업로드, 만료일 자동 상태 관리 (valid/expired/expiring_soon)
- **문제**: 이것도 플랫폼 관리자 전용. 학원이 직접 올리고 관리해야 함
- **문제**: 차량 5대면 차량별 검사증, 기사별 면허/범죄경력... 관리 항목이 많은데 차량/기사와 문서를 연결하는 기능 없음 (전부 학원 단위로만 관리)
- **요청**: 차량별, 기사별 문서 관리 + 만료 전 자동 알림 (이메일/앱 푸시)
- **요청**: 문서 갱신 이력 (언제 바뀌었는지 추적)

#### 4. 권한 관리 / 역할 분리 [점수: 2/5]
- 역할이 parent, driver, escort, admin(academy_admin), platform_admin으로 잘 나뉨
- **문제**: 학원 내부에서 원장/실장/사무직 같은 하위 역할이 없음. 300명 학원이면 실장이 학생 관리, 원장이 청구 관리를 나눠서 하고 싶음
- **문제**: 감사 로그(PlatformAuditLogPage)가 플랫폼 관리자 전용. 학원에서도 "누가 언제 뭘 변경했는지" 볼 수 있어야 함
- **요청**: 학원 내 세부 권한 설정 (읽기전용, 학생관리만, 청구관리만 등)

#### 5. 운행 분석/리포트 [점수: 1.5/5]
- 기본 KPI (학생수, 차량수, 오늘스케줄, 미결제)만 있음
- **없는 것들**:
  - 차량별 운행 통계 (일/주/월)
  - 정시 도착률
  - 차량 가동률 (5대 중 실제 운행 차량)
  - 기사별 운행 실적
  - 학생별 탑승 이력
  - 월별 운영 보고서 (PDF 출력)
- **요청**: 경영진 보고용 월간 보고서 자동 생성

#### 6. 관제센터 / 실시간 모니터링 [점수: 3.5/5 (플랫폼 관리자 기준)]
- PlatformMapPage가 잘 만들어져 있음: 카카오맵 연동, 5초 간격 GPS 폴링, 차량 목록 사이드바, 검색, 상태 표시(운행중/대기/비활성)
- API 키 없을 때 테이블 폴백도 있음
- **하지만 학원 관리자가 접근 불가** → 이건 치명적
- **요청**: 학원 관리자용 관제 화면 (자기 학원 차량만)

### 이수진 실장 종합 의견

**기존 전산 시스템 대비**: 우리 ERP와 직접 연동이 안 되면 이중 입력이 됨. API는 있지만 서버간 인증(API 키)이 없고 Webhook도 없어서 실시간 동기화 불가. 결국 수동으로 두 시스템을 관리해야 함.

**컴플라이언스**: 기능 자체는 잘 갖춰져 있는데 학원이 직접 관리할 수 없는 게 문제. 차량 5대, 기사 5명이면 보험증서, 검사증, 면허 등 최소 15개 문서를 관리해야 하는데, 매번 플랫폼 관리자에게 요청해야 함.

**대규모 운영 적합성**: 300명 + 5대 차량이면 매일 아침 배차만 해도 복잡한데, 스케줄 템플릿 설정 UI가 없고 학생별 반복 스케줄을 만들 수 없어서 자동화의 핵심이 빠져 있음. 파이프라인 버튼은 있지만 그 전에 템플릿을 만드는 과정이 없으니 무용지물.

**비용 대비 가치**: 현재 상태로는 도입 근거 부족. ERP 연동 가능, 서버사이드 페이지네이션, 학원 내 권한 관리, 운행 리포트가 추가되어야 우리 규모에서 도입을 검토할 수 있음.

**총점: 45/100**

## 결론

세이프웨이키즈는 **백엔드 아키텍처와 데이터 모델이 탄탄하고, UI 디자인도 현대적**입니다. 하지만 현재 가장 큰 문제는 **"플랫폼 관리자 vs 학원 관리자" 기능 배분의 불균형**입니다.

핵심 가치 기능(관제센터, 엑셀 업로드, 컴플라이언스)이 모두 플랫폼 관리자 전용이고, 학원 관리자는 학생 CRUD + 차량 CRUD + 청구 관리 정도만 할 수 있습니다. 스케줄 템플릿 관리 UI가 없어서 "자동 배차"라는 핵심 기능이 사실상 사용 불가능한 상태입니다.

**지금 바로 해결해야 서비스 출시가 가능한 것**: 스케줄 템플릿 UI, 관제센터 학원 개방, 엑셀 업로드 학원 개방, 스케줄/청구서에 학생 이름 표시.

이 4가지만 해결되면 중소 학원(30~100명)을 대상으로 베타 서비스가 가능합니다.
