# 운전기사 페르소나 평가: Edge AI PoC 데모 계획서

**평가일:** 2026-03-27
**평가자:** 김기사 (가명) — 어린이 통학차량 운전기사, 경력 8년
**평가 대상:** SafeWay Kids Edge AI PoC Requirement Brief + Consensus Matrix
**배경:** 58세, 하루 5회 운행(오전 2회/오후 3회), 회당 15~20명, 스마트폰 기본 수준

---

## 1. 운전 방해도 (20점 만점)

### 점수: 16 / 20

**좋은 점:**
- 시나리오 ②(이상행동 감지)에서 경고가 화면에 빨간 박스랑 배너로 뜬다고 하니, 소리만 적절하면 운전 중에 크게 방해는 안 될 것 같다.
- 사각지대 감지(시나리오 ④)가 출발 전/주차 때만 동작한다면 운행 중에는 신경 안 써도 되니까 괜찮다.
- 음성 경고를 사전 녹음 MP3로 한다니까, 기계 목소리보다 자연스럽고 듣기 편할 것 같다.

**걱정되는 점:**
- 운행 중에 이상행동 감지 경고음이 너무 자주 울리면 진짜 위험할 때 "또 오작동이겠지" 하고 무시하게 된다. 경고음 빈도 조절이 꼭 필요하다.
- 경고가 뜰 때 운전석에서 어떻게 알 수 있는지가 안 나와 있다. 화면은 운전 중에 못 보는데, 소리만으로 충분한가? 진동 알림이나 운전석 전용 표시등 같은 게 있으면 좋겠다.
- 운행 중에 AI가 계속 카메라로 분석하고 있으면, 혹시 화면이 번쩍거리거나 해서 백미러로 보일 때 신경 쓰이지 않을지 걱정된다.

**개선 필요:**
- 경고 등급별 소리 구분이 필요하다. "주의"는 짧은 알림음, "위험"은 반복 경고음 같은 식으로.
- 운전석에서 확인 가능한 방법(스피커 위치, 소리 크기 조절)을 명시해야 한다.

## 2. 잔류 인원 확인 도움 (20점 만점)

### 점수: 18 / 20

**좋은 점:**
- 이거 정말 필요하다. 매번 운행 끝나면 뒤에서부터 좌석 하나하나 확인하고, 화장실 문도 열어보고... 특히 겨울에 아이가 이불 덮고 자고 있으면 못 보고 지나칠 뻔한 적도 있다.
- "시동 OFF" 버튼 누르면 자동으로 스캔한다는 게 좋다. 내가 별도로 뭘 조작할 필요 없이 시동만 끄면 되니까.
- 사람이 없으면 "ALL CLEAR" 초록색으로 뜨고, 있으면 빨간색 + 음성 경고라니 직관적이다.
- 잔류 인원 감지 정확도 99% 목표라는데, 실제로 이 정도면 진짜 안심이 된다.

**걱정되는 점:**
- 아이가 좌석 밑에 숨어 있거나, 담요 덮고 누워 있을 때도 감지가 되는가? 카메라 각도에 따라 사각지대가 있을 수 있다.
- 웹캠 1대로는 통학버스 뒤쪽 좌석까지 다 커버가 안 될 것 같다. 실제 버스는 좌석이 10줄 넘는데.
- "ALL CLEAR" 떴는데 실제로 아이가 있으면 큰일이다. 이런 경우(미감지)에 대한 안전장치가 필요하다.

**개선 필요:**
- 실제 차량에서는 카메라 여러 대가 필요하다는 점을 데모에서 설명해야 한다. PoC라서 1대지만 실제는 3~4대 설치한다는 식으로.
- 잔류 인원 스캔을 여러 번 반복 확인(예: 3회 연속 스캔)하는 로직이 있으면 더 안심된다. 문서에 "multi-frame 분석"이라고는 있는데 구체적으로 몇 프레임인지가 궁금하다.

## 3. 사각지대 감지 도움 (20점 만점)

### 점수: 17 / 20

**좋은 점:**
- 차량 주변 조감도(Bird's Eye View)로 어디에 아이가 있는지 한눈에 보이는 건 정말 유용하다. 지금은 사이드미러랑 룸미러만 보는데 한계가 있다.
- 3단계 경고(주의/경고/위험)를 색깔로 구분하는 것이 직관적이다. 노란색-주황색-빨간색이면 바로 알 수 있다.
- 출발 전에 차량 주변 확인할 때 이 시스템이 있으면 아이가 차 뒤에 있는 걸 놓칠 확률이 확 줄어든다.
- 경고 거리를 실제 센서 범위에 맞춰서 조정했다는 게 현실적이다(초음파 4m 범위 내에서 3.5m/2m/0.8m).

**걱정되는 점:**
- 이건 시뮬레이션이라 실제 차량에 센서를 달아봐야 진짜 효과를 알 수 있다. 데모에서는 좋아 보여도 실제 도로에서 비가 오거나 눈이 오면 센서가 제대로 작동할지 모르겠다.
- 주차장에서 출발할 때 아이들이 여러 방향에서 동시에 접근하면 어떻게 되나? 복수 어린이 시나리오가 있다고는 하는데 5명 이상은 어떤가.
- 운전석 모니터에 조감도를 항상 띄워놓을 수 있는 건지, 아니면 별도 장치가 필요한 건지.

**개선 필요:**
- 실제 차량 적용 시 모니터 위치/크기에 대한 고려가 필요하다.
- 악천후(비, 눈, 안개) 상황에서의 센서 성능 저하에 대한 언급이 있으면 좋겠다.

## 4. 사용 용이성 (20점 만점)

### 점수: 17 / 20

**좋은 점:**
- 원클릭 실행이라는 점이 제일 좋다. 컴퓨터 잘 모르는 사람도 파일 하나 더블클릭하면 된다니까.
- 시동 OFF하면 자동으로 잔류 인원 검사 시작하는 것처럼, 내가 따로 뭘 할 필요가 없는 자동 동작이 핵심이다.
- 한국어 UI라서 영어 메뉴 때문에 헤맬 일이 없다.
- 백엔드 없어도 독립 실행(standalone) 되는 것이 좋다. 인터넷이 안 되는 시골 노선에서도 동작해야 하니까.

**걱정되는 점:**
- 처음 설치할 때 Python 설치, pip install 이런 걸 해야 하나? 나는 이런 거 못 한다. 기사들 대부분 못 한다.
- 시스템이 먹통이 됐을 때 재시작하는 방법이 간단해야 한다. "껐다 켜기" 수준으로.
- 얼굴 등록을 기사가 직접 해야 하나? 학원 선생님이 미리 해놓는 건가? 매번 새 학생 올 때마다 등록해야 하면 번거롭다.
- 데모 계획서에 "스텝 위저드"라는 게 나오는데, 이건 데모용이고 실제 운행에서는 시나리오 전환이 자동이어야 한다.

**개선 필요:**
- 설치를 원클릭 인스톨러(exe 파일)로 만들어야 현장 배포가 가능하다.
- 얼굴 등록은 학원 관리자가 웹에서 미리 하고, 차량 시스템에 자동 동기화되는 구조가 필요하다.
- 시스템 오류 시 자동 재시작 기능이 있어야 한다.

## 5. 신뢰성/오작동 (20점 만점)

### 점수: 14 / 20

**좋은 점:**
- 안면인식을 ArcFace ONNX로 바꿔서 정확도 99.83%라는데, 이 정도면 거짓 인식은 거의 없을 것 같다.
- 5프레임 연속 확인(temporal smoothing)으로 한 번 잘못 감지한 걸로 바로 경고 안 뜨게 한다는 것이 현실적이다. 카메라가 흔들릴 때 오작동을 줄여줄 수 있다.
- 모델 warm-up으로 시작할 때 오작동을 줄인다는 것도 괜찮다.

**걱정되는 점:**
- **이게 제일 중요하다.** 거짓 경고가 하루에 10번 넘으면 일주일 안에 소리를 꺼버린다. 보조교사도 마찬가지다. "또 울리네" 하고 무시하게 되면 진짜 위험할 때도 반응 안 하게 된다.
- 이상행동 감지에서, 아이가 가방을 꺼내려고 잠깐 일어서는 것과 진짜 위험한 행동을 구분할 수 있나? 아이들은 수시로 몸을 움직인다. 차가 출발하기 전에 서 있는 건 정상인데 이것도 경고가 뜨나?
- 차량 진동이 심한 비포장도로에서 카메라 흔들림이 오감지를 유발하지 않을지.
- 겨울에 아이들이 두꺼운 패딩 입고 모자 쓰고 마스크하면 안면인식이 제대로 되나? 마스크 착용 시 70%라고 했는데 이건 너무 낮다. 10명 중 3명을 못 알아보면 매번 수동 확인해야 한다.
- FPS가 3~4밖에 안 된다고 했는데, 느린 반응 때문에 경고가 늦게 뜨면 소용이 없다.

**개선 필요:**
- **이상행동 감지 민감도 조절 기능이 필수다.** 운행 중/정차 중 모드를 구분해서, 정차 중에는 서 있어도 경고 안 뜨게 해야 한다.
- 거짓 경고 발생률 목표치를 명시해야 한다. 하루 5회 운행 기준 거짓 경고 3회 이내 정도.
- 마스크+모자 착용 시 인식률을 높일 방안이 필요하다. 못 알아보면 "수동 확인 필요" 알림이라도 띄워야 한다.
- 진동/흔들림 보정 로직이 있는지 확인 필요.

## 운전기사로서 반드시 물을 질문 5개

1. **"거짓 경고가 하루에 몇 번이나 울립니까?"**
   — 하루 5회 운행하는데, 매 운행마다 거짓 경고가 2~3번씩 울리면 못 쓴다. 구체적인 숫자를 알고 싶다.

2. **"아이가 잠깐 일어서서 가방 꺼내는 것도 경고가 뜹니까? 출발 전에 서 있는 것도요?"**
   — 현장에서는 아이들이 자리에서 일어나는 게 흔한 일이다. 차가 멈춰 있을 때는 당연히 서 있고, 이런 것까지 경고가 뜨면 무시하게 된다.

3. **"겨울에 아이들이 패딩 모자 마스크 다 쓰고 타는데, 얼굴을 못 알아보면 어떻게 합니까?"**
   — 11월~3월은 거의 다 완전무장이다. 이때 인식 못 하면 결국 내가 수동으로 출석 체크해야 한다. 그러면 이 시스템이 왜 필요한가.

4. **"이 기계가 먹통 되면 어떻게 합니까? 나는 컴퓨터 수리를 못 합니다."**
   — 아침 첫 운행인데 시스템이 안 켜지면 그냥 출발해야 한다. 수리 기사 올 때까지 기다릴 수 없다. 자동으로 다시 켜지거나, 최소한 전화 한 통이면 원격으로 고쳐줄 수 있어야 한다.

5. **"내가 직접 설정하거나 조작해야 하는 게 있습니까? 솔직히 그런 건 못 합니다."**
   — 얼굴 등록, 시나리오 전환, 설정 변경 같은 건 학원 선생님이나 본사에서 다 해놓고, 나는 시동 걸면 자동으로 돌아가는 것만 원한다.

## 최종 평가

### 총점: 82 / 100

| 기준 | 점수 | 비고 |
|------|------|------|
| 운전 방해도 | 16/20 | 운전석 알림 방식 보완 필요 |
| 잔류 인원 확인 도움 | 18/20 | 가장 실용적, 사각지대 커버리지만 보완 |
| 사각지대 감지 도움 | 17/20 | 유용하나 실제 센서 적용 전까지는 시뮬레이션 |
| 사용 용이성 | 17/20 | 자동 동작 좋으나 설치/등록 과정 간소화 필요 |
| 신뢰성/오작동 | 14/20 | 거짓 경고 억제가 가장 큰 과제 |

### 종합 의견

> **"잔류 인원 감지는 당장이라도 있었으면 좋겠다. 매번 뒤에서 앞까지 걸어가면서 좌석 하나하나 확인하는 게 겨울에는 특히 힘들다. 시동 끄면 자동으로 확인해주고 ALL CLEAR 뜨면 그때 내리면 되니까, 이건 진짜 편해질 것 같다."**

> **"근데 이상행동 감지는 솔직히 걱정이다. 아이들이 차에서 얌전히 앉아만 있나? 가방 꺼내고, 친구랑 장난치고, 일어서서 창밖 보고... 이런 게 다 경고로 뜨면 하루에 몇십 번은 울릴 거다. 한 달이면 소리를 꺼버린다. 그러면 진짜 위험할 때도 못 듣는다. 이 부분을 꼭 해결해야 한다."**

> **"사각지대 감지도 있으면 좋겠지만, 이건 실제로 센서를 차에 달아봐야 안다. 화면으로 보는 시뮬레이션은 심사위원한테 보여주기에는 괜찮겠지만, 현장에서 진짜 쓸 수 있는지는 아직 모르겠다."**

> **"결론적으로, 잔류 인원 감지 + 사각지대 감지는 '이게 있으면 일이 편해지겠다'이고, 이상행동 감지는 거짓 경고 문제를 해결하면 쓸 만하겠지만 지금 상태로는 '좀 더 다듬어야 된다'이다. 전체적으로 방향은 맞다. 기사들한테 불편 안 주면서 안전을 도와주는 도구가 되면 환영한다."**
