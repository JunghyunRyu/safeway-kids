## 학원A: 정은서 원장 (태권도 학원, 80명, 2대) 재검증

### 이전 총점: 55/100

### 개선된 항목

1. **스케줄 템플릿 UI (B01)** — 탭 기반 UI가 추가되어 반복 스케줄 조회/활성화/비활성화가 가능해졌습니다. 이전에는 "파이프라인"만 있고 템플릿이 없었는데, 이제 학생별 요일/시간/주소가 보입니다. **다만** 템플릿 **신규 생성** 기능은 안 보입니다. 기존 템플릿을 보고 토글만 할 수 있지, "월~금 오후 3시 김민수 학생 픽업" 같은 새 템플릿을 추가하는 폼이 없습니다. 이건 반쪽짜리 해결입니다.

2. **관제센터 학원 개방 (B02)** — 학원 관리자 사이드바에 "관제 센터" 메뉴가 추가되었습니다. PlatformMapPage를 공유하는 방식인데, 학원 관리자가 접근하면 **모든 차량**이 보이는 것 아닌가요? `PlatformMapPage`에서 `/telemetry/vehicles` API를 호출하는데, 학원 소속 차량만 필터링하는 로직이 없습니다. 백엔드에서 RBAC로 걸러주는지 확인 필요합니다.

3. **보호자 연락처 (A06)** — 학생 등록 시 보호자 전화번호 입력/전송이 가능해졌습니다. 엑셀 업로드에서도 "보호자전화번호" 열을 인식합니다. 이건 완전히 해결되었습니다.

| N2 | 일일 스케줄 테이블에 여전히 student_id UUID 앞 8글자만 표시 | **Medium** | 미해결 잔류 |

핵심 개선이 이루어졌습니다. 스케줄 템플릿이 보이고, 관제센터에 접근 가능하고, 보호자 연락처가 작동합니다. **하지만** 템플릿을 "보는 것"과 "만드는 것"은 다릅니다. 80명 학생의 반복 스케줄을 누가 어떻게 처음 만들어 넣는지가 불분명합니다. 그리고 일일 스케줄에서 여전히 학생 이름이 안 보이는 건 일상 운영에서 불편합니다.

### 베타 서비스 출시 가능 여부

**중소 학원(30~100명) 베타**: 조건부 가능

조건:
1. N1 해결 (스케줄 템플릿 신규 생성 폼) — 없으면 핵심 워크플로우 불가
2. N2 해결 (일일 스케줄 학생 이름 표시) — 일상 운영에 필수
3. N3 해결 (관제센터 학원 차량 필터) — 타 학원 차량이 보이면 보안/프라이버시 문제
4. N4 해결 (사이드바에 엑셀 업로드 메뉴 추가) — 기능이 있어도 찾지 못하면 없는 것

위 4건 해결 시 중소 학원 대상 제한적 베타 가능.

**대형 학원(300명+)**: 불가 — 서버사이드 페이지네이션, 학원 내 역할 분리, ERP 연동 필요.

**프랜차이즈 본사 도입**: 불가 — 가격 모델 재설계, 규제 승인, 파일럿 데이터 필요 (코드 영역 밖).
