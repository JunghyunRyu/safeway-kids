# Milestone 3 Report — Real-time Features Hardening

**Date**: 2026-03-13
**Status**: COMPLETE

## Scope
Harden all real-time features: GPS tracking with real device sensors, push notifications via FCM, Kakao Maps rendering in WebView, and WebSocket reconnection with exponential backoff.

## What Was Delivered

### 1. Kakao Maps Integration
- Full HTML asset (`kakao-map.html`) with dynamic API key loading
- Supports bus markers, stop markers (numbered, color-coded), polylines, ETA overlay, driver position
- Bidirectional communication via `postMessage` / `injectJavaScript`

### 2. Parent MapScreen (Live Tracking)
- `useVehicleTracking` hook manages WebSocket connections to multiple vehicles
- Exponential backoff reconnection: 3s → 6s → 12s → max 30s
- Bus positions rendered as blue circle markers on Kakao Maps
- ETA shown as "N정거장 전" based on remaining stops
- Connection status indicator (green/red dot)

### 3. Driver MapScreen (GPS + Stops)
- Real GPS via `expo-location` with `Accuracy.High`
- Driver position pushed to server every 5s via `useGpsTracking`
- Stop markers from daily schedule displayed on map
- GPS status indicator (green/gray dot)

### 4. Push Notifications
- `expo-notifications` with FCM token auto-registration on login
- Foreground notification display with sound and badge
- Android notification channel with MAX importance
- Backend fire-and-forget push on board/alight events (from M2)

### 5. Permissions
- `app.json` configured with iOS plist strings (Korean) and Android permissions
- Location permission request with Korean alert on denial
