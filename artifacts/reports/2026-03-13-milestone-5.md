# Milestone 5 Report — Operational Loop Completion

**Date**: 2026-03-13
**Status**: COMPLETE

## Scope
Complete the operational loop: automated schedule materialization, VRP-TW route generation, driver route display, Kakao road distance integration, and one-command demo seeding.

## What Was Delivered

### 1. Daily Pipeline Orchestrator
- `scheduler.py`: materialize → auto-assign vehicles → VRP-TW route generation
- Three entry points:
  - **API**: `POST /schedules/daily/pipeline?target_date=YYYY-MM-DD` (admin)
  - **CLI**: `python -m app.modules.scheduling --date YYYY-MM-DD`
  - **Programmatic**: `run_daily_pipeline(db, date)`
- Auto vehicle assignment: round-robin active vehicles to active drivers
- Auto route generation: VRP-TW solver runs per academy for the target date

### 2. Driver Route Display
- `GET /routes/my-route?plan_date=YYYY-MM-DD` — driver-specific endpoint
- **RouteScreen**: Shows stops in VRP-TW optimized order with info banner ("AI 최적화 노선 v1, 2.3km · 약 12분")
- **MapScreen**: Renders route-ordered stop markers on Kakao Maps
- Falls back to pickup-time order when no route plan exists

### 3. Kakao Road Distance Integration
- `distance.py`: builds distance/time matrix by calling Kakao Mobility API for all stop pairs
- Redis caching with 24h TTL (avoids repeated API calls for same coordinates)
- Graceful fallback to Euclidean distance when API is unavailable or key is missing
- Solver accepts pre-computed matrices for road distance accuracy

### 4. Seed Data CLI
- `python -m app.seed` creates a complete demo dataset:
  - 1 academy (강남 코딩학원, Gangnam coordinates)
  - 2 vehicles (15-seat), 2 drivers
  - 3 parents, 5 students with enrollments and consents
  - Schedule templates (Mon-Fri, 2:00-2:40 PM)
  - Vehicle assignments for today + tomorrow
  - Runs daily pipeline automatically after seeding
- Idempotent: skips if data already exists

## Operational Flow (End-to-End)
```
Seed/Admin Setup
  → Schedule Templates (weekly recurring)
    → Daily Pipeline (materialize + assign + route)
      → Driver App: fetches optimized route
        → Driver drives: GPS → POST /telemetry/gps → Redis
          → Parent App: WebSocket receives GPS → bus on map
            → Driver: board/alight → push notification to parent
```
